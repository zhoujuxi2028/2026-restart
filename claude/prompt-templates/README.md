### 5. 测试用例生成模板
**文件**: `test-case-template.md`
**用途**: 请求Claude生成测试用例

## 🎯 如何使用模板

### 步骤1：选择模板
根据您的需求，选择合适的模板文件。

### 步骤2：填充变量
模板中用 `[方括号]` 标记的部分需要您填写：
- `[技术栈]` → 填入实际技术栈，如"Cypress"
- `[代码]` → 粘贴实际代码
- `[问题描述]` → 描述您的具体问题

### 步骤3：调整细节
根据具体情况，调整模板内容：
- 添加约束条件
- 调整审查维度
- 指定输出格式

### 步骤4：使用模板
复制填充好的模板，在与Claude对话时使用。

## 🔄 模板维护

### 何时创建新模板
- 发现自己经常问类似问题
- 某种提问方式特别有效
- 需要复杂的多维度分析

### 如何改进模板
1. 使用后记录效果
2. 根据反馈调整
3. 添加新的约束条件
4. 优化输出格式要求

### 模板版本控制
每次改进模板时：
- 记录改动原因
- 对比改进前后效果
- 保留有效的版本

## 💡 最佳实践

### ✅ DO（推荐做法）
1. **先选模板，再填充**：不要从零开始写Prompt
2. **保留成功案例**：好的对话记录下来，提取为模板
3. **持续优化**：每次使用后思考如何改进
4. **分类管理**：按场景分类，方便查找

### ❌ DON'T（避免做法）
1. **不要死板套用**：根据具体情况灵活调整
2. **不要过度复杂**：模板太长反而降低效率
3. **不要忽略反馈**：记录使用效果，持续改进
4. **不要只收藏不用**：模板的价值在于使用

## 🎓 模板进阶技巧

### 技巧1：组合使用
将多个模板组合，处理复杂场景：
```
第1轮：用学习模板了解概念
第2轮：用功能开发模板实现
第3轮：用测试模板写测试
第4轮：用代码审查模板Review
```

### 技巧2：个性化定制
根据自己的习惯定制模板：
- 添加常用的约束条件
- 指定偏好的输出格式
- 加入项目特定要求

### 技巧3：建立模板索引
创建快速参考卡片，方便查找：
```
需要审查代码 → code-review-template.md
遇到bug → debug-template.md
开发新功能 → feature-development-template.md
```
