```
测试文件：cypress/e2e/login.cy.js
功能：用户登录测试

对比记录：
之前的方式：一次性要求"写一个完整的登录测试"
可能包含不需要的功能、难以调整、失去控制权

现在的方式：分3轮迭代完成
第1轮：基本结构（describe + it）
第2轮：添加访问页面（cy.visit）
第3轮：添加输入和点击操作

哪种方式更好：迭代式对话

为什么：
1. 可控性：每一步都在掌控中
2. 灵活性：随时可以调整方向
3. 理解性：逐步理解代码构建过程
4. 质量高：每步验证，减少返工
```

**今日收获**：
```
最大的收获：
学会了迭代式对话，不再一次性要求完整功能

遇到的挑战：
需要改变思维习惯，习惯分步骤思考

明天改进：
在实际项目中应用迭代式对话
```

1. 目标：要达到什么效果
2. 约束：有什么限制（时间、技术、格式）
3. 上下文：相关的背景信息
4. 预期：期望的输出是什么
