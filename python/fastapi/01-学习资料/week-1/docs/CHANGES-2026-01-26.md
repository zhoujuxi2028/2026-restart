## 📝 主要改进内容

### 代码层面

**Before (旧代码)**:
```python
if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000, reload=True)  # ❌ 产生警告
```

**After (新代码)**:
```python
"""
运行方式1 (学习推荐):
    python3 基础概念/day2-exercises.py

运行方式2 (开发推荐):
    uvicorn 基础概念.day2-exercises:app --reload --port 8000

区别:
- 方式1: 简单直接，适合学习，但修改代码后需手动重启
- 方式2: 支持热重载，修改代码自动生效，适合开发

访问: http://localhost:8000/docs 查看API文档
"""
if __name__ == "__main__":
    import uvicorn
    # 使用 reload=False 避免警告信息
    # 如需热重载功能，请使用上面的"运行方式2"
    uvicorn.run(app, host="0.0.0.0", port=8000, reload=False)  # ✅ 无警告
```

### 文档层面

1. **运行方式清晰说明**
   - 方式1: Python 直接运行（推荐学习）
   - 方式2: Uvicorn 命令（推荐开发）
   - 详细对比两种方式的优缺点

2. **常见问题扩展**

## 💡 经验教训

### 技术层面

1. **ASGI 服务器的 reload 机制**
   - reload=True 需要字符串形式的 app 引用
   - 直接传递 app 对象无法实现热重载
   - 学习材料应该考虑初学者体验

2. **两种运行方式的权衡**
   - Python 直接运行：简单但功能有限
   - Uvicorn 命令：强大但学习曲线陡峭
   - 应该根据学习阶段选择合适方式

### 教学设计层面

1. **渐进式学习很重要**
   - 初学者不应该被工具配置困扰
   - 应该先专注于核心概念
   - 工具的使用可以后续逐步引入

2. **文档的重要性**
   - 清晰的注释减少困惑
   - 详细的说明文档避免重复问题
   - 故障排除指南提高学习效率

3. **错误信息的影响**
   - 即使不影响功能，警告也会让初学者不安
   - 应该尽量提供清爽的学习体验
   - 预防问题比解决问题更重要
