## 第 1 题：Selenium 有哪几种等待机制？它们有什么区别?

**答案**：

Selenium 提供三种等待机制：

| 等待类型 | 作用范围 | 优点 | 缺点 | 推荐度 |
|---------|---------|------|------|-------|
| **强制等待** | 指定代码行 | 简单直接 | 浪费时间，不够智能 | ❌ 不推荐 |
| **隐式等待** | 全局所有元素查找 | 配置一次全局生效 | 可能增加测试时间，与显式等待冲突 | ⚠️ 谨慎使用 |
| **显式等待** | 特定元素或条件 | 灵活智能，按需等待 | 代码稍多 | ✅ **强烈推荐** |

**代码示例**：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
import time

driver = webdriver.Chrome()

# 1. 强制等待（不推荐）
time.sleep(3)  # 固定等待3秒，无论元素是否加载完成

# 2. 隐式等待（全局设置）
driver.implicitly_wait(10)  # 所有 find_element 操作最多等待10秒
element = driver.find_element(By.ID, "username")  # 如果元素不存在，等待最多10秒

# 3. 显式等待（推荐）
wait = WebDriverWait(driver, 10)  # 最多等待10秒
element = wait.until(
    EC.presence_of_element_located((By.ID, "username"))
)
```

**关键点**：
- 强制等待会固定浪费时间，应避免使用
- 隐式等待设置后影响所有元素查找，可能与显式等待冲突
- 显式等待是最佳实践，按需等待，灵活高效

## 第 2 题：显式等待的常用条件有哪些？

**答案**：

Selenium 提供了 `expected_conditions` 模块，包含丰富的等待条件。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Chrome()
wait = WebDriverWait(driver, 10)

# 1. presence_of_element_located - 元素存在于 DOM 中（可能不可见）
element = wait.until(
    EC.presence_of_element_located((By.ID, "username"))
)

# 2. visibility_of_element_located - 元素可见（存在且 display 不为 none）
element = wait.until(
    EC.visibility_of_element_located((By.ID, "message"))
)

# 3. element_to_be_clickable - 元素可点击（可见且可用）
button = wait.until(
    EC.element_to_be_clickable((By.ID, "submit"))
)

# 4. invisibility_of_element_located - 元素不可见或不存在
wait.until(
    EC.invisibility_of_element_located((By.ID, "loading"))
)

# 5. text_to_be_present_in_element - 元素包含指定文本
wait.until(
    EC.text_to_be_present_in_element((By.ID, "status"), "成功")
)

# 6. text_to_be_present_in_element_value - 元素 value 属性包含指定文本
wait.until(
    EC.text_to_be_present_in_element_value((By.ID, "input"), "test")
)

# 7. frame_to_be_available_and_switch_to_it - frame 可用并切换
wait.until(
    EC.frame_to_be_available_and_switch_to_it((By.ID, "myframe"))
)

# 8. alert_is_present - 警告框存在
wait.until(EC.alert_is_present())

# 9. element_to_be_selected - 元素被选中（checkbox/radio）
wait.until(
    EC.element_to_be_selected(driver.find_element(By.ID, "checkbox"))
)

# 10. staleness_of - 元素不再附加到 DOM（已移除）
old_element = driver.find_element(By.ID, "dynamic")
driver.find_element(By.ID, "refresh").click()
wait.until(EC.staleness_of(old_element))

# 11. number_of_windows_to_be - 窗口数量达到指定值
wait.until(EC.number_of_windows_to_be(2))

# 12. new_window_is_opened - 新窗口已打开
wait.until(EC.new_window_is_opened(driver.window_handles))

# 13. title_is - 页面标题完全匹配
wait.until(EC.title_is("首页"))

# 14. title_contains - 页面标题包含指定文本
wait.until(EC.title_contains("登录"))

# 15. url_contains - URL 包含指定文本
wait.until(EC.url_contains("/dashboard"))

# 16. url_to_be - URL 完全匹配
wait.until(EC.url_to_be("https://example.com/home"))

# 17. url_changes - URL 发生变化
current_url = driver.current_url
driver.find_element(By.ID, "link").click()
wait.until(EC.url_changes(current_url))

# 18. presence_of_all_elements_located - 至少有一个元素存在
elements = wait.until(
    EC.presence_of_all_elements_located((By.CLASS_NAME, "item"))
)

# 19. visibility_of_any_elements_located - 至少有一个元素可见
elements = wait.until(
    EC.visibility_of_any_elements_located((By.CLASS_NAME, "item"))
)

# 20. visibility_of_all_elements_located - 所有元素都可见
elements = wait.until(
    EC.visibility_of_all_elements_located((By.CLASS_NAME, "item"))
)
```

**常用条件分类**：

| 类别 | 条件 | 使用场景 |
|------|------|---------|
| **元素存在** | `presence_of_element_located` | 元素出现在 DOM 中 |
| **元素可见** | `visibility_of_element_located` | 元素可见并可操作 |
| **元素可点击** | `element_to_be_clickable` | 点击操作前 |
| **元素消失** | `invisibility_of_element_located` | 加载动画消失 |
| **文本出现** | `text_to_be_present_in_element` | 验证消息显示 |
| **页面跳转** | `url_contains`, `title_contains` | 等待页面加载完成 |
| **弹窗处理** | `alert_is_present` | Alert 出现 |
| **窗口切换** | `number_of_windows_to_be` | 等待新窗口打开 |

**关键点**：
- 选择合适的等待条件可以提高测试稳定性
- `element_to_be_clickable` 是最常用的条件
- 等待条件可以避免 `NoSuchElementException`

## 第 3 题：如何自定义等待条件？

**答案**：

当内置等待条件不满足需求时，可以自定义等待条件。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Chrome()
wait = WebDriverWait(driver, 10)

# 方法 1：使用 lambda 函数（简单条件）
# 等待元素的 class 包含 "active"
wait.until(
    lambda d: "active" in d.find_element(By.ID, "menu").get_attribute("class")
)

# 等待元素数量达到 5 个
wait.until(
    lambda d: len(d.find_elements(By.CLASS_NAME, "item")) == 5
)

# 方法 2：定义自定义等待类（复杂条件）
class element_has_css_class:
    """等待元素包含指定 CSS class"""

    def __init__(self, locator, css_class):
        self.locator = locator
        self.css_class = css_class

    def __call__(self, driver):
        element = driver.find_element(*self.locator)
        classes = element.get_attribute("class")
        if classes and self.css_class in classes:
            return element
        return False

# 使用自定义条件
wait.until(
    element_has_css_class((By.ID, "menu"), "active")
)

# 方法 3：等待元素属性变化
class element_attribute_to_be:
    """等待元素属性变为指定值"""

    def __init__(self, locator, attribute, value):
        self.locator = locator
        self.attribute = attribute
        self.value = value

    def __call__(self, driver):
        element = driver.find_element(*self.locator)
        attr_value = element.get_attribute(self.attribute)
        return attr_value == self.value

# 等待 input 的 value 属性变为 "loaded"
wait.until(
    element_attribute_to_be((By.ID, "status"), "value", "loaded")
)

# 方法 4：等待 JavaScript 条件
class js_condition_true:
    """等待 JavaScript 表达式返回 true"""

    def __init__(self, js_script):
        self.js_script = js_script

    def __call__(self, driver):
        return driver.execute_script(f"return {self.js_script}")

# 等待 jQuery 加载完成
wait.until(
    js_condition_true("typeof jQuery !== 'undefined'")
)

# 等待 AJAX 请求完成
wait.until(
    js_condition_true("jQuery.active == 0")
)

# 等待 Angular 加载完成
wait.until(
    js_condition_true("window.getAllAngularTestabilities().findIndex(x => !x.isStable()) === -1")
)

# 方法 5：等待元素数量变化
class element_count_to_be:
    """等待元素数量达到指定值"""

    def __init__(self, locator, count):
        self.locator = locator
        self.count = count

    def __call__(self, driver):
        elements = driver.find_elements(*self.locator)
        if len(elements) == self.count:
            return elements
        return False

# 等待列表项增加到 10 个
wait.until(
    element_count_to_be((By.CLASS_NAME, "list-item"), 10)
)

# 方法 6：等待元素样式变化
class element_style_to_contain:
    """等待元素 style 属性包含指定内容"""

    def __init__(self, locator, style_property, expected_value):
        self.locator = locator
        self.style_property = style_property
        self.expected_value = expected_value

    def __call__(self, driver):
        element = driver.find_element(*self.locator)
        style_value = element.value_of_css_property(self.style_property)
        return self.expected_value in style_value

# 等待元素 display 样式变为 block
wait.until(
    element_style_to_contain((By.ID, "modal"), "display", "block")
)

# 方法 7：等待页面加载状态
class page_ready_state:
    """等待页面加载完成"""

    def __call__(self, driver):
        return driver.execute_script("return document.readyState") == "complete"

wait.until(page_ready_state())

# 方法 8：组合多个条件（全部满足）
class all_conditions_met:
    """等待多个条件全部满足"""

    def __init__(self, *conditions):
        self.conditions = conditions

    def __call__(self, driver):
        for condition in self.conditions:
            if not condition(driver):
                return False
        return True

# 等待多个元素都可见
wait.until(
    all_conditions_met(
        EC.visibility_of_element_located((By.ID, "header")),
        EC.visibility_of_element_located((By.ID, "footer")),
        EC.element_to_be_clickable((By.ID, "submit"))
    )
)

# 方法 9：等待条件中的任意一个满足
class any_condition_met:
    """等待任意一个条件满足"""

    def __init__(self, *conditions):
        self.conditions = conditions

    def __call__(self, driver):
        for condition in self.conditions:
            try:
                result = condition(driver)
                if result:
                    return result
            except:
                pass
        return False

# 等待成功或失败消息（任意一个出现）
result = wait.until(
    any_condition_met(
        EC.visibility_of_element_located((By.ID, "success-message")),
        EC.visibility_of_element_located((By.ID, "error-message"))
    )
)
```

**实用自定义条件库**：

```python
class CustomWaitConditions:
    """自定义等待条件集合"""

    @staticmethod
    def element_count_greater_than(locator, count):
        """元素数量大于指定值"""
        def _predicate(driver):
            elements = driver.find_elements(*locator)
            return elements if len(elements) > count else False
        return _predicate

    @staticmethod
    def element_text_contains(locator, text):
        """元素文本包含指定内容"""
        def _predicate(driver):
            element = driver.find_element(*locator)
            return text in element.text
        return _predicate

    @staticmethod
    def ajax_complete():
        """AJAX 请求完成"""
        def _predicate(driver):
            return driver.execute_script("return jQuery.active == 0")
        return _predicate

    @staticmethod
    def file_download_complete(download_dir, filename):
        """文件下载完成"""
        import os
        def _predicate(driver):
            file_path = os.path.join(download_dir, filename)
            return os.path.exists(file_path)
        return _predicate

# 使用
wait.until(CustomWaitConditions.element_count_greater_than((By.CLASS_NAME, "item"), 5))
```

**关键点**：
- 自定义条件需要返回 True/False 或元素对象
- lambda 适合简单条件，类适合复杂可复用条件
- 可以组合多个条件提高灵活性

## 第 4 题：如何处理等待超时的情况？

**答案**：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.common.exceptions import TimeoutException
import logging

driver = webdriver.Chrome()
logging.basicConfig(level=logging.INFO)

# 方法 1：基本异常捕获
try:
    wait = WebDriverWait(driver, 10)
    element = wait.until(
        EC.presence_of_element_located((By.ID, "username"))
    )
except TimeoutException:
    print("等待超时：元素未在 10 秒内出现")
    driver.save_screenshot("timeout_screenshot.png")

# 方法 2：自定义超时消息
try:
    wait = WebDriverWait(driver, 10)
    element = wait.until(
        EC.element_to_be_clickable((By.ID, "submit")),
        message="提交按钮在 10 秒内未变为可点击状态"
    )
except TimeoutException as e:
    print(f"超时错误: {e}")

# 方法 3：超时后的降级策略
def find_element_with_fallback(driver, locators, timeout=10):
    """尝试多个定位器，超时则尝试下一个"""
    wait = WebDriverWait(driver, timeout)

    for by, value in locators:
        try:
            element = wait.until(
                EC.presence_of_element_located((by, value))
            )
            return element
        except TimeoutException:
            logging.warning(f"定位器超时: {by}={value}")
            continue

    raise TimeoutException("所有定位器都超时")

# 使用
locators = [
    (By.ID, "username"),
    (By.NAME, "username"),
    (By.CSS_SELECTOR, "input[type='text']")
]
element = find_element_with_fallback(driver, locators)

# 方法 4：超时后重试
def wait_with_retry(driver, condition, timeout=10, max_retries=3):
    """超时后自动重试"""
    wait = WebDriverWait(driver, timeout)

    for attempt in range(max_retries):
        try:
            return wait.until(condition)
        except TimeoutException:
            if attempt < max_retries - 1:
                logging.warning(f"等待超时，重试 {attempt + 1}/{max_retries}")
                driver.refresh()  # 刷新页面重试
            else:
                logging.error("达到最大重试次数，等待失败")
                raise

# 使用
element = wait_with_retry(
    driver,
    EC.presence_of_element_located((By.ID, "username"))
)

# 方法 5：超时后执行清理操作
class WaitWithCleanup:
    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout

    def wait_for(self, condition, cleanup_action=None):
        wait = WebDriverWait(self.driver, self.timeout)
        try:
            return wait.until(condition)
        except TimeoutException:
            logging.error("等待超时，执行清理操作")
            if cleanup_action:
                cleanup_action(self.driver)
            raise

# 使用
def cleanup(driver):
    driver.delete_all_cookies()
    driver.refresh()

waiter = WaitWithCleanup(driver, timeout=10)
element = waiter.wait_for(
    EC.presence_of_element_located((By.ID, "username")),
    cleanup_action=cleanup
)

# 方法 6：动态调整超时时间
class AdaptiveWait:
    def __init__(self, driver, base_timeout=10):
        self.driver = driver
        self.base_timeout = base_timeout

    def wait_for(self, condition, timeout_multiplier=1):
        """根据条件重要性调整超时时间"""
        adjusted_timeout = self.base_timeout * timeout_multiplier
        wait = WebDriverWait(self.driver, adjusted_timeout)

        try:
            return wait.until(condition)
        except TimeoutException:
            logging.error(f"等待超时（{adjusted_timeout}秒）")
            raise

# 使用
adaptive_wait = AdaptiveWait(driver, base_timeout=10)

# 普通元素：10秒
element1 = adaptive_wait.wait_for(
    EC.presence_of_element_located((By.ID, "username")),
    timeout_multiplier=1
)

# 重要元素：30秒
element2 = adaptive_wait.wait_for(
    EC.presence_of_element_located((By.ID, "payment-button")),
    timeout_multiplier=3
)

# 方法 7：超时后的详细诊断
def wait_with_diagnosis(driver, locator, timeout=10):
    """超时后提供详细诊断信息"""
    wait = WebDriverWait(driver, timeout)

    try:
        element = wait.until(
            EC.presence_of_element_located(locator)
        )
        return element
    except TimeoutException:
        by, value = locator
        diagnosis = {
            "url": driver.current_url,
            "title": driver.title,
            "page_source_length": len(driver.page_source),
            "locator": f"{by}={value}",
            "screenshot": "timeout.png"
        }

        # 检查元素是否在 DOM 中
        elements = driver.find_elements(*locator)
        diagnosis["element_count"] = len(elements)

        # 检查页面加载状态
        ready_state = driver.execute_script("return document.readyState")
        diagnosis["ready_state"] = ready_state

        # 保存截图
        driver.save_screenshot(diagnosis["screenshot"])

        logging.error(f"等待超时诊断信息: {diagnosis}")
        raise

# 使用
wait_with_diagnosis(driver, (By.ID, "username"))

# 方法 8：超时后发送通知
def wait_with_notification(driver, condition, timeout=10, notify_func=None):
    """超时后发送通知（如邮件、Slack）"""
    wait = WebDriverWait(driver, timeout)

    try:
        return wait.until(condition)
    except TimeoutException as e:
        error_msg = f"等待超时: {condition}"
        if notify_func:
            notify_func(error_msg)
        raise

# 使用
def send_alert(message):
    # 发送邮件或 Slack 通知
    print(f"发送告警: {message}")

wait_with_notification(
    driver,
    EC.presence_of_element_located((By.ID, "username")),
    notify_func=send_alert
)
```

**超时处理最佳实践**：

```python
class RobustWait:
    """健壮的等待机制"""

    def __init__(self, driver, default_timeout=10):
        self.driver = driver
        self.default_timeout = default_timeout
        self.logger = logging.getLogger(__name__)

    def wait_for_element(self, locator, timeout=None, retries=1):
        """等待元素出现（支持重试）"""
        timeout = timeout or self.default_timeout

        for attempt in range(retries):
            try:
                wait = WebDriverWait(self.driver, timeout)
                element = wait.until(
                    EC.presence_of_element_located(locator)
                )
                return element
            except TimeoutException:
                if attempt < retries - 1:
                    self.logger.warning(f"等待超时，重试 {attempt + 1}/{retries}")
                    self.driver.refresh()
                else:
                    self._handle_timeout(locator)
                    raise

    def _handle_timeout(self, locator):
        """超时后的处理逻辑"""
        self.logger.error(f"等待超时: {locator}")

        # 截图
        screenshot_path = f"timeout_{self._get_timestamp()}.png"
        self.driver.save_screenshot(screenshot_path)

        # 记录页面信息
        self.logger.error(f"当前 URL: {self.driver.current_url}")
        self.logger.error(f"页面标题: {self.driver.title}")

        # 保存页面源码
        with open(f"timeout_{self._get_timestamp()}.html", "w") as f:
            f.write(self.driver.page_source)

    @staticmethod
    def _get_timestamp():
        from datetime import datetime
        return datetime.now().strftime("%Y%m%d_%H%M%S")

# 使用
robust_wait = RobustWait(driver)
element = robust_wait.wait_for_element((By.ID, "username"), timeout=15, retries=3)
```

**关键点**：
- 超时异常应该捕获并记录详细信息
- 截图和保存页面源码有助于问题诊断
- 考虑重试机制处理间歇性问题
- 超时时间应根据实际情况调整

## 第 5 题：隐式等待和显式等待可以混用吗？为什么？

**答案**：

**不推荐混用**，因为会导致不可预测的等待时间和行为。

**问题原因**：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
import time

driver = webdriver.Chrome()

# 设置隐式等待 10 秒
driver.implicitly_wait(10)

# 使用显式等待 5 秒
wait = WebDriverWait(driver, 5)

# 问题：实际等待时间可能是 10 + 5 = 15 秒！
try:
    element = wait.until(
        EC.presence_of_element_located((By.ID, "not-exist"))
    )
except:
    print("元素未找到")
    # 可能等待了 15 秒而不是预期的 5 秒
```

**问题分析**：

| 场景 | 隐式等待 | 显式等待 | 实际等待时间 |
|------|---------|---------|------------|
| 元素不存在 | 10秒 | 5秒 | **15秒**（相加） |
| 元素 3 秒后出现 | 10秒 | 5秒 | 3秒（正常） |
| 元素 8 秒后出现 | 10秒 | 5秒 | 8秒（但可能触发超时） |

**正确做法**：

```python
# 方法 1：只使用显式等待（推荐）
driver = webdriver.Chrome()
# 不设置隐式等待
wait = WebDriverWait(driver, 10)
element = wait.until(
    EC.presence_of_element_located((By.ID, "username"))
)

# 方法 2：如果必须使用隐式等待，不要混用显式等待
driver = webdriver.Chrome()
driver.implicitly_wait(10)
# 直接使用 find_element，依赖隐式等待
element = driver.find_element(By.ID, "username")
```

## 第 6 题：如何等待页面完全加载（包括 AJAX 和动态内容）？

```python
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# 方法 1：等待 document.readyState 为 complete
def wait_for_page_load(driver, timeout=30):
    wait = WebDriverWait(driver, timeout)

    wait.until(lambda d: d.execute_script("return jQuery.active == 0"))

class PageLoadWaiter:
    def __init__(self, driver, timeout=30):
        self.driver = driver
        self.timeout = timeout

    def wait_for_full_load(self):
        """等待页面完全加载"""
        wait = WebDriverWait(self.driver, self.timeout)

        # 1. 等待 DOM 加载完成
        wait.until(lambda d: d.execute_script("return document.readyState") == "complete")

        # 2. 等待 jQuery AJAX（如果页面使用 jQuery）
        jquery_loaded = self.driver.execute_script("return typeof jQuery !== 'undefined'")
        if jquery_loaded:
            wait.until(lambda d: d.execute_script("return jQuery.active == 0"))

        # 3. 等待网络空闲（没有新的请求）
        self._wait_for_network_idle()

    def _wait_for_network_idle(self, idle_time=0.5):
        """等待网络空闲（无新请求）"""
        import time
        last_request_count = self._get_request_count()
        time.sleep(idle_time)
        current_request_count = self._get_request_count()

        while current_request_count > last_request_count:
            last_request_count = current_request_count
            time.sleep(idle_time)
            current_request_count = self._get_request_count()

    def _get_request_count(self):
        """获取当前请求数量（Chrome only）"""
        try:
            return self.driver.execute_script("""
                return performance.getEntriesByType('resource').length;
            """)
        except:
            return 0

# 使用

waiter.wait_for_full_load()

def wait_for_api_response(driver, api_url, timeout=30):
    """等待特定 API 请求完成"""
    wait = WebDriverWait(driver, timeout)

    def check_api_response(d):
        logs = d.get_log('performance')
        for log in logs:
            import json
            message = json.loads(log['message'])['message']
            if message['method'] == 'Network.responseReceived':
                response = message['params']['response']
                if api_url in response['url']:
                    return True
        return False

    wait.until(check_api_response)

    """等待页面准备就绪（业务角度）"""
    wait = WebDriverWait(driver, 10)

    wait.until(EC.presence_of_element_located((By.ID, "main-content")))

    # 等待加载动画消失
    wait.until(EC.invisibility_of_element_located((By.CLASS_NAME, "loading")))
```

| 策略 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| `document.readyState` | 简单 | 不等待 AJAX | 静态页面 |
| `jQuery.active` | 准确 | 依赖 jQuery | jQuery 项目 |

- 组合多种策略提高可靠性

## 第 7 题：如何设置等待的轮询频率（poll_frequency）？

**答案**：

`WebDriverWait` 默认每 0.5 秒检查一次条件，可以通过 `poll_frequency` 参数调整。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.common.exceptions import NoSuchElementException
import time

driver.get("https://example.com")

    EC.presence_of_element_located((By.ID, "username"))
)

# 自定义轮询频率：每 0.1 秒检查一次（更频繁）
wait = WebDriverWait(driver, timeout=10, poll_frequency=0.1)
element = wait.until(
    EC.element_to_be_clickable((By.ID, "submit"))
)

# 降低轮询频率：每 2 秒检查一次（减少资源消耗）
wait = WebDriverWait(driver, timeout=30, poll_frequency=2)
element = wait.until(
    EC.presence_of_element_located((By.ID, "slow-loading-element"))
)

# 忽略特定异常
wait = WebDriverWait(
    driver,
    timeout=10,
    poll_frequency=0.5,
    ignored_exceptions=[NoSuchElementException]  # 忽略元素未找到异常
)
element = wait.until(
    lambda d: d.find_element(By.ID, "username")

def performance_test():
    # 测试 1：高频率轮询（0.1 秒）
    start = time.time()
    wait_fast = WebDriverWait(driver, 5, poll_frequency=0.1)
    try:
        wait_fast.until(EC.presence_of_element_located((By.ID, "not-exist")))
    except:
        fast_time = time.time() - start
        print(f"高频率轮询：{fast_time:.2f}秒，约检查 {fast_time / 0.1:.0f} 次")

    # 测试 2：低频率轮询（1 秒）
    start = time.time()
    wait_slow = WebDriverWait(driver, 5, poll_frequency=1)
    try:
        wait_slow.until(EC.presence_of_element_located((By.ID, "not-exist")))
    except:
        slow_time = time.time() - start
        print(f"低频率轮询：{slow_time:.2f}秒，约检查 {slow_time / 1:.0f} 次")

# 根据场景选择轮询频率
class AdaptivePollingWait:
    """自适应轮询频率"""

    def __init__(self, driver):
        self.driver = driver

    def wait_for_fast_element(self, locator, timeout=10):
        """快速出现的元素（如按钮点击后的反馈）"""
        wait = WebDriverWait(self.driver, timeout, poll_frequency=0.1)
        return wait.until(EC.presence_of_element_located(locator))

    def wait_for_slow_element(self, locator, timeout=30):
        """缓慢加载的元素（如大数据表格）"""
        wait = WebDriverWait(self.driver, timeout, poll_frequency=1)
        return wait.until(EC.presence_of_element_located(locator))

    def wait_for_normal_element(self, locator, timeout=10):
        """普通元素（使用默认频率）"""
        wait = WebDriverWait(self.driver, timeout)  # 默认 0.5 秒
        return wait.until(EC.presence_of_element_located(locator))

# 使用
adaptive_wait = AdaptivePollingWait(driver)

# 快速元素（如提示消息）
message = adaptive_wait.wait_for_fast_element((By.ID, "toast-message"))

# 慢速元素（如大型报表）
report = adaptive_wait.wait_for_slow_element((By.ID, "data-report"))

# 自定义轮询逻辑
class CustomPollingWait:
    """自定义轮询策略（递增间隔）"""

    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout

    def wait_with_backoff(self, condition):
        """使用指数退避策略"""
        start_time = time.time()
        poll_interval = 0.1  # 初始间隔 0.1 秒

        while time.time() - start_time < self.timeout:
            try:
                result = condition(self.driver)
                if result:
                    return result
            except:
                pass

            time.sleep(poll_interval)
            # 每次检查后增加间隔（最大 2 秒）
            poll_interval = min(poll_interval * 1.5, 2)

        raise TimeoutException("等待超时")

# 使用
custom_wait = CustomPollingWait(driver)
element = custom_wait.wait_with_backoff(
    lambda d: d.find_element(By.ID, "username")
)
```

**轮询频率选择指南**：

| 场景 | 推荐频率 | 原因 |
|------|---------|------|
| **快速反馈元素**（提示、Toast） | 0.1 - 0.2秒 | 及时捕获短暂元素 |
| **普通元素** | 0.5秒（默认） | 平衡性能和响应 |
| **慢速加载元素**（报表、大数据） | 1 - 2秒 | 减少不必要的检查 |
| **资源受限环境**（CI/CD） | 1秒 | 降低 CPU 占用 |

**最佳实践**：

```python
class SmartWait:
    """智能等待（自动选择轮询频率）"""

    def __init__(self, driver, default_timeout=10):
        self.driver = driver
        self.default_timeout = default_timeout

    def wait_for_element(self, locator, timeout=None, element_type="normal"):
        """根据元素类型选择轮询频率"""
        timeout = timeout or self.default_timeout

        # 根据元素类型选择轮询频率
        poll_frequency = {
            "fast": 0.1,      # 快速元素
            "normal": 0.5,    # 普通元素
            "slow": 1.0,      # 慢速元素
            "very_slow": 2.0  # 非常慢的元素
        }.get(element_type, 0.5)

        wait = WebDriverWait(
            self.driver,
            timeout=timeout,
            poll_frequency=poll_frequency
        )

        return wait.until(EC.presence_of_element_located(locator))

# 使用

# 快速元素

# 慢速元素

```

- 默认轮询频率 0.5 秒适合大多数场景
- 高频率轮询（0.1秒）适合捕获短暂元素
- 低频率轮询（1-2秒）适合慢速加载场景

```python
    def wait_for_attribute_change(self, locator, attribute, expected_value):
        """等待属性变为指定值"""
        self.wait.until(
            lambda d: d.find_element(*locator).get_attribute(attribute) == expected_value
        )

    def wait_for_style_change(self, locator, style_property, expected_value):
        """等待样式变化"""
        self.wait.until(
            lambda d: d.find_element(*locator).value_of_css_property(style_property) == expected_value
```

```python
class WaitChain:
    """等待链（记录每个步骤）"""

    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout
        self.logger = logging.getLogger(__name__)
        self.steps = []

    def add_step(self, condition, description):
        """添加等待步骤"""
        self.steps.append((condition, description))
        return self

    def execute(self):
        """执行所有步骤"""
        self.logger.info(f"开始执行等待链 (共 {len(self.steps)} 步)")

        for i, (condition, description) in enumerate(self.steps, 1):
            self.logger.info(f"步骤 {i}/{len(self.steps)}: {description}")
            start = time.time()

            try:
                wait = WebDriverWait(self.driver, self.timeout)
                wait.until(condition)
                elapsed = time.time() - start
                self.logger.info(f"✓ 步骤 {i} 完成 (耗时 {elapsed:.2f}秒)")
            except TimeoutException:
                self.logger.error(f"✗ 步骤 {i} 失败: {description}")
                raise

        self.logger.info("等待链执行完成")
```

```python
# 方法 1：重试机制
def wait_with_retry(driver, condition, timeout=10, retries=3):
    """等待失败时自动重试"""
    for attempt in range(retries):
        try:
            wait = WebDriverWait(driver, timeout)
            return wait.until(condition)
        except TimeoutException:
            if attempt < retries - 1:
                print(f"等待失败，重试 {attempt + 1}/{retries}")
                driver.refresh()
                time.sleep(1)
            else:
                raise

# 使用

# 方法 2：等待元素稳定（不再变化）
def wait_for_element_stable(driver, locator, timeout=10, stable_time=2):
    """等待元素状态稳定"""
    wait = WebDriverWait(driver, timeout)

    # 先等待元素出现
    element = wait.until(EC.presence_of_element_located(locator))

    # 记录初始状态
    last_state = element.get_attribute("outerHTML")
    stable_start = time.time()

    while time.time() - stable_start < stable_time:
        time.sleep(0.5)
        try:
            current_state = element.get_attribute("outerHTML")
            if current_state != last_state:
                # 状态变化，重置计时
                last_state = current_state
                stable_start = time.time()
        except StaleElementReferenceException:
            # 元素过时，重新定位
            element = driver.find_element(*locator)
            last_state = element.get_attribute("outerHTML")
            stable_start = time.time()

    return element

# 使用

    driver,
    (By.ID, "dynamic-content"),
    stable_time=2  # 等待元素 2 秒内不变化
)

def safe_interact(driver, locator, action, max_attempts=3):

    for attempt in range(max_attempts):
        try:
            element = driver.find_element(*locator)
            action(element)
            return
        except StaleElementReferenceException:
            if attempt < max_attempts - 1:
                print(f"元素过时，重试 {attempt + 1}/{max_attempts}")
                time.sleep(0.5)

                raise

class PersistentElementWaiter:
    """等待元素持续可见"""

    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout

    def wait_for_persistent_visibility(self, locator, duration=2):
        """等待元素持续可见指定时间"""
        wait = WebDriverWait(self.driver, self.timeout)

        # 等待元素首次可见
        element = wait.until(EC.visibility_of_element_located(locator))

        # 验证持续可见
        start = time.time()
        while time.time() - start < duration:
            try:
                if not element.is_displayed():
                    # 元素消失，重新等待
                    element = wait.until(EC.visibility_of_element_located(locator))
                    start = time.time()
            except StaleElementReferenceException:
                element = wait.until(EC.visibility_of_element_located(locator))
                start = time.time()

            time.sleep(0.1)

        return element

def poll_until_success(driver, action, timeout=10, interval=0.5):
    """轮询直到操作成功"""
    start = time.time()

    while time.time() - start < timeout:
        try:
            return action(driver)
        except Exception as e:
            print(f"操作失败: {e}，继续重试...")
            time.sleep(interval)

    raise TimeoutException("轮询超时")
```

```python
class UnstableElementHandler:
    """不稳定元素处理器"""

    def __init__(self, driver, timeout=10, max_retries=3):
        self.driver = driver
        self.timeout = timeout
        self.max_retries = max_retries

    def find_and_click(self, locator):
        """查找并点击不稳定元素"""
        return self._retry_on_stale(
            locator,
            lambda el: el.click()
        )

    def find_and_send_keys(self, locator, text):
        """查找并输入不稳定元素"""
        return self._retry_on_stale(
            locator,
            lambda el: el.send_keys(text)
        )

    def get_text(self, locator):
        """获取不稳定元素文本"""
        return self._retry_on_stale(
            locator,
            lambda el: el.text
        )

    def _retry_on_stale(self, locator, action):
        """过时时重试"""
        wait = WebDriverWait(self.driver, self.timeout)

        for attempt in range(self.max_retries):
            try:
                element = wait.until(
                    EC.presence_of_element_located(locator)
                )
                return action(element)
            except StaleElementReferenceException:
                if attempt < self.max_retries - 1:
                    print(f"元素过时，重试 {attempt + 1}/{self.max_retries}")
                    time.sleep(0.5)
                else:
                    raise

# 使用

handler.find_and_click((By.ID, "unstable-button"))
text = handler.get_text((By.ID, "unstable-text"))
```

- 捕获 `StaleElementReferenceException` 并重新定位
- 使用 `ignored_exceptions` 参数忽略间歇性异常

## 第 11 题：如何在 CI/CD 环境中优化等待策略？

**答案**：

CI/CD 环境通常资源有限、网络不稳定，需要特殊的等待策略。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
import os

# 检测是否在 CI 环境
def is_ci_environment():
    """检测是否在 CI/CD 环境"""
    ci_indicators = [
        'CI', 'CONTINUOUS_INTEGRATION',
        'JENKINS_URL', 'GITHUB_ACTIONS',
        'GITLAB_CI', 'CIRCLECI',
        'TRAVIS'
    ]
    return any(os.getenv(var) for var in ci_indicators)

# 根据环境调整超时时间
class EnvironmentAwareWait:
    """环境感知的等待策略"""

    def __init__(self, driver):
        self.driver = driver
        self.is_ci = is_ci_environment()

        # CI 环境使用更长的超时时间
        self.default_timeout = 30 if self.is_ci else 10
        self.poll_frequency = 1 if self.is_ci else 0.5

        print(f"环境: {'CI' if self.is_ci else '本地'}, "
              f"超时: {self.default_timeout}秒, "
              f"轮询: {self.poll_frequency}秒")

    def wait_for_element(self, locator, timeout=None):
        """等待元素出现"""
        timeout = timeout or self.default_timeout
        wait = WebDriverWait(
            self.driver,
            timeout=timeout,
            poll_frequency=self.poll_frequency
        )
        return wait.until(EC.presence_of_element_located(locator))

# 使用

# CI 环境专用配置
class CIOptimizedWait:
    """CI 环境优化的等待策略"""

    def __init__(self, driver):
        self.driver = driver
        self.is_ci = is_ci_environment()

    def wait_for_page_load(self):
        """等待页面加载（CI 环境更宽容）"""
        timeout = 60 if self.is_ci else 30
        wait = WebDriverWait(self.driver, timeout)

        # CI 环境只等待 DOM 就绪
        if self.is_ci:
            wait.until(
                lambda d: d.execute_script("return document.readyState") == "complete"
            )
        else:
            # 本地环境等待更详细
            wait.until(
                lambda d: d.execute_script("return document.readyState") == "complete"
            )
            # 等待 AJAX（如果使用 jQuery）
            try:
                wait.until(
                    lambda d: d.execute_script("return jQuery.active == 0")
                )
            except:
                pass

    def wait_for_element_clickable(self, locator):
        """等待元素可点击（CI 环境增加重试）"""
        timeout = 30 if self.is_ci else 10
        max_retries = 3 if self.is_ci else 1

        for attempt in range(max_retries):
            try:
                wait = WebDriverWait(self.driver, timeout)
                return wait.until(EC.element_to_be_clickable(locator))
            except:
                if attempt < max_retries - 1:
                    print(f"CI: 等待失败，重试 {attempt + 1}/{max_retries}")
                    self.driver.refresh()
                else:
                    raise

# 使用
```

```python
# 技巧 3：缓存元素定位结果
class CachedElementWaiter:
    """缓存元素定位结果"""

    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout
        self.cache = {}

    def get_element(self, locator, cache_key=None):
        """获取元素（带缓存）"""
        cache_key = cache_key or str(locator)

        # 检查缓存
        if cache_key in self.cache:
            try:
                element = self.cache[cache_key]
                # 验证元素仍然有效
                element.is_displayed()
                return element
            except:
                # 元素过时，删除缓存
                del self.cache[cache_key]

        # 重新定位并缓存
        wait = WebDriverWait(self.driver, self.timeout)
        element = wait.until(EC.presence_of_element_located(locator))
        self.cache[cache_key] = element
        return element

# 技巧 7：提前终止等待
class EarlyTerminationWait:
    """支持提前终止的等待"""

    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout
        self.should_stop = False

    def wait_until(self, condition, stop_condition=None):
        """等待条件，或遇到停止条件时提前结束"""
        start = time.time()

        while time.time() - start < self.timeout:
            # 检查停止条件
            if stop_condition and stop_condition(self.driver):
                raise Exception("遇到停止条件，提前终止")

            # 检查等待条件
            try:
                result = condition(self.driver)
                if result:
                    return result
            except:
                pass

            time.sleep(0.5)

        raise TimeoutException("等待超时")

# 技巧 10：性能监控和优化
class PerformanceOptimizedWait:
    """性能优化的等待类"""

    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout
        self.stats = {"total_time": 0, "count": 0}

    def wait_for(self, condition):
        start = time.time()
        wait = WebDriverWait(self.driver, self.timeout)
        result = wait.until(condition)
        elapsed = time.time() - start

        # 记录统计
        self.stats["total_time"] += elapsed
        self.stats["count"] += 1

        # 如果等待时间过长，记录警告
        if elapsed > self.timeout * 0.8:
            print(f"警告：等待时间接近超时 ({elapsed:.2f}秒)")

        return result

    def get_average_wait_time(self):
        if self.stats["count"] == 0:
            return 0
        return self.stats["total_time"] / self.stats["count"]

# 使用

perf_wait.wait_for(EC.presence_of_element_located((By.ID, "element2")))
print(f"平均等待时间: {perf_wait.get_average_wait_time():.2f}秒")
```
