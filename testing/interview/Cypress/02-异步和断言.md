## 1. Cypress 如何处理异步操作？为什么不需要 async/await？

**答案：**
Cypress 命令是异步的但不返回 Promise，而是返回 Chainable 对象，命令会自动排队执行。

**核心原理：**
- 所有 Cypress 命令被添加到命令队列
- 命令按顺序自动执行
- 内置智能等待和重试机制
- 不需要手动处理异步

**代码示例：**
```javascript
// ❌ 错误：尝试使用 async/await
it('错误示例', async () => {
  await cy.get('[data-cy="username"]').type('user')  // 不需要 await
  const text = await cy.get('[data-cy="title"]').text()  // 无法直接获取值
})

// ✅ 正确：Cypress 链式调用
it('正确示例', () => {
  cy.get('[data-cy="username"]').type('user')
  cy.get('[data-cy="title"]').should('contain', '欢迎')
  // 命令自动排队，按顺序执行
})

// 命令执行顺序示例
it('命令队列', () => {
  console.log('1. 测试开始')  // 立即执行

  cy.visit('/')                // 加入队列
  cy.get('[data-cy="btn"]')    // 加入队列
  cy.click()                   // 加入队列

  console.log('2. 命令已排队')  // 立即执行（在 Cypress 命令之前）

  // 实际执行顺序：
  // 1. 测试开始 → 2. 命令已排队 → visit → get → click
})
```

## 2. 如何在 Cypress 中获取和使用元素的值？

**答案：**
使用 `.then()` 处理前一个命令的结果，不能直接赋值。

**代码示例：**
```javascript
// ❌ 错误：无法直接获取值
const text = cy.get('[data-cy="title"]').text()  // 返回 Chainable，不是文本
console.log(text)  // 无法打印实际值

// ✅ 方法 1：使用 .then() 处理值
cy.get('[data-cy="title"]').then(($el) => {
  const text = $el.text()  // jQuery 对象
  expect(text).to.include('欢迎')
  console.log(text)  // 可以打印
})

// ✅ 方法 2：使用 invoke() 调用方法
cy.get('[data-cy="title"]')
  .invoke('text')
  .then((text) => {
    expect(text).to.include('欢迎')
  })

// ✅ 方法 3：使用 alias 保存值
cy.get('[data-cy="username"]')
  .invoke('text')
  .as('username')  // 保存为 alias

cy.get('@username').then((username) => {
  cy.log(`当前用户: ${username}`)
})

// 实际应用：比较两个元素的值
cy.get('[data-cy="price"]')
  .invoke('text')
  .then((priceText) => {
    const price = parseFloat(priceText.replace('¥', ''))
    expect(price).to.be.greaterThan(100)
  })
```

## 3. Cypress 的自动等待机制是如何工作的？

**答案：**
Cypress 会自动等待元素出现、可见、可交互，直到满足条件或超时（默认 4 秒）。

**自动等待场景：**
- 元素存在于 DOM 中
- 元素可见（不是 hidden）
- 元素不被覆盖
- 元素未被禁用
- 动画完成

**代码示例：**
```javascript
// Cypress 会自动等待元素出现
cy.get('[data-cy="submit"]').click()
// 内部逻辑：
// 1. 查找元素（最多等待 4 秒）
// 2. 检查元素可见性
// 3. 检查元素不被覆盖
// 4. 执行点击

// 自定义超时时间
cy.get('[data-cy="slow-element"]', { timeout: 10000 })
  .should('be.visible')

// should() 会自动重试
cy.get('[data-cy="counter"]')
  .should('have.text', '10')  // 会重试直到文本为 '10' 或超时

// 等待元素出现并消失
cy.get('[data-cy="loading"]').should('be.visible')
cy.get('[data-cy="loading"]').should('not.exist')
cy.get('[data-cy="content"]').should('be.visible')

// 实际应用：等待异步加载的内容
cy.intercept('GET', '/api/users').as('getUsers')
cy.visit('/users')
cy.wait('@getUsers')  // 等待请求完成
cy.get('[data-cy="user-list"]')
  .find('[data-cy="user-item"]')
  .should('have.length.greaterThan', 0)  // 自动重试直到有数据
```

## 4. 如何使用断言？`should()` 和 `expect()` 有什么区别？

**答案：**
- `should()`：链式断言，自动重试，推荐用于元素
- `expect()`：BDD 风格断言，不重试，用于普通值

**代码示例：**
```javascript
// should() - 链式断言（推荐）
cy.get('[data-cy="title"]')
  .should('be.visible')
  .should('contain', '欢迎')
  .should('have.class', 'active')

// should() 支持链式组合
cy.get('[data-cy="input"]')
  .should('be.visible')
  .and('have.value', '')  // and() 是 should() 的别名
  .and('have.attr', 'placeholder', '请输入用户名')

// expect() - BDD 断言（不重试）
cy.get('[data-cy="price"]').then(($el) => {
  const price = parseFloat($el.text())
  expect(price).to.be.greaterThan(100)
  expect(price).to.be.lessThan(1000)
})

// 常用断言
cy.get('[data-cy="button"]')
  .should('be.visible')          // 可见
  .should('be.enabled')          // 启用
  .should('have.text', '提交')   // 文本内容
  .should('have.class', 'btn')   // CSS 类
  .should('have.attr', 'type', 'submit')  // 属性
  .should('have.css', 'color', 'rgb(255, 0, 0)')  // CSS 样式

// 否定断言
cy.get('[data-cy="error"]').should('not.exist')
cy.get('[data-cy="button"]').should('not.be.disabled')

// 自定义断言回调
cy.get('[data-cy="list"]')
  .find('li')
  .should(($lis) => {
    expect($lis).to.have.length(3)
    expect($lis.eq(0)).to.contain('第一项')
    expect($lis.eq(1)).to.contain('第二项')
  })
```

## 5. 如何测试 URL 和导航？

**答案：**
使用 `cy.url()` 和 `cy.location()` 获取 URL 信息并断言。

**代码示例：**
```javascript
// 断言当前 URL
cy.url().should('include', '/dashboard')
cy.url().should('eq', 'http://localhost:3000/dashboard')

// 断言 URL 部分
cy.location('pathname').should('eq', '/users/123')
cy.location('search').should('eq', '?page=1')
cy.location('hash').should('eq', '#section-1')

// 完整 location 对象
cy.location().should((loc) => {
  expect(loc.protocol).to.eq('http:')
  expect(loc.hostname).to.eq('localhost')
  expect(loc.port).to.eq('3000')
  expect(loc.pathname).to.eq('/dashboard')
})

// 导航测试
it('应该正确导航', () => {
  cy.visit('/')
  cy.get('[data-cy="login-link"]').click()
  cy.url().should('include', '/login')

  cy.go('back')  // 后退
  cy.url().should('eq', 'http://localhost:3000/')

  cy.go('forward')  // 前进
  cy.url().should('include', '/login')

  cy.reload()  // 刷新页面
})

// 实际应用：测试查询参数
cy.visit('/search?q=cypress&category=testing')
cy.location('search').should('include', 'q=cypress')
cy.location('search').should('include', 'category=testing')
```

## 6. 如何组织测试？`describe()` 和 `it()` 的作用是什么？

**答案：**
- `describe()`：测试套件（Test Suite），组织相关测试
- `it()`：测试用例（Test Case），单个测试
- `context()`：`describe()` 的别名，用于不同上下文

**代码示例：**
```javascript
// 基本结构
describe('用户管理', () => {
  it('应该显示用户列表', () => {
    cy.visit('/users')
    cy.get('[data-cy="user-list"]').should('be.visible')
  })

  it('应该能够添加新用户', () => {
    cy.visit('/users/new')
    cy.get('[data-cy="name"]').type('张三')
    cy.get('[data-cy="submit"]').click()
  })
})

// 嵌套组织
describe('购物车功能', () => {
  context('空购物车', () => {
    it('应该显示空状态提示', () => {
      cy.visit('/cart')
      cy.contains('购物车为空').should('be.visible')
    })
  })

  context('有商品的购物车', () => {
    beforeEach(() => {
      // 添加测试数据
      cy.addItemToCart('product-1')
    })

    it('应该显示商品数量', () => {
      cy.visit('/cart')
      cy.get('[data-cy="cart-count"]').should('contain', '1')
    })

    it('应该能够删除商品', () => {
      cy.visit('/cart')
      cy.get('[data-cy="remove-btn"]').click()
      cy.contains('购物车为空').should('be.visible')
    })
  })
})

// 测试标签
describe('登录功能', () => {
  it('应该成功登录', { tags: ['@smoke', '@auth'] }, () => {
    // 测试代码
  })

  it('应该处理错误', { tags: ['@error'] }, () => {
    // 测试代码
  })
})
```

## 7. 钩子函数有哪些？如何使用？

**答案：**
钩子函数用于在测试前后执行代码，常用于初始化和清理。

**钩子类型：**
- `before()`：所有测试前执行一次
- `beforeEach()`：每个测试前执行
- `after()`：所有测试后执行一次
- `afterEach()`：每个测试后执行

**代码示例：**
```javascript
describe('用户管理', () => {
  // 所有测试前执行一次
  before(() => {
    cy.log('开始测试用户管理功能')
    // 初始化测试数据库
  })

  // 每个测试前执行
  beforeEach(() => {
    cy.visit('/users')
    // 登录
    cy.login('admin@example.com', 'password')
  })

  // 每个测试后执行
  afterEach(() => {
    // 清理测试数据
    cy.task('clearDatabase')
  })

  // 所有测试后执行一次
  after(() => {
    cy.log('用户管理功能测试完成')
  })

  it('应该显示用户列表', () => {
    cy.get('[data-cy="user-list"]').should('be.visible')
  })

  it('应该能够编辑用户', () => {
    cy.get('[data-cy="edit-btn"]').first().click()
  })
})

// 实际应用：跨测试共享数据
describe('表单提交', () => {
  let testData

  before(() => {
    // 加载测试数据
    cy.fixture('users.json').then((data) => {
      testData = data
    })
  })

  beforeEach(() => {
    cy.visit('/form')
  })

  it('应该验证必填字段', () => {
    cy.get('[data-cy="submit"]').click()
    cy.contains('请填写必填字段').should('be.visible')
  })

  it('应该成功提交', () => {
    cy.get('[data-cy="name"]').type(testData.name)
    cy.get('[data-cy="email"]').type(testData.email)
    cy.get('[data-cy="submit"]').click()
    cy.contains('提交成功').should('be.visible')
  })
})
```

## 8. 如何使用 `.then()` 和 `.wrap()` 处理异步值？

**答案：**
`.then()` 处理前一个命令的结果，`.wrap()` 将普通值包装为 Cypress 命令。

**代码示例：**
```javascript
// .then() 处理命令结果
cy.get('[data-cy="count"]')
  .invoke('text')
  .then((text) => {
    const count = parseInt(text)
    cy.log(`当前计数: ${count}`)

    // 在 then 中可以使用 if 判断
    if (count > 10) {
      cy.get('[data-cy="reset"]').click()
    }
  })

// .wrap() 将值包装回 Cypress 链
cy.get('[data-cy="prices"]')
  .then(($prices) => {
    const prices = $prices.toArray().map(el => parseFloat(el.innerText))
    const total = prices.reduce((sum, price) => sum + price, 0)
    return cy.wrap(total)  // 包装为 Cypress 命令
  })
  .should('be.greaterThan', 100)

// 实际应用：条件逻辑
cy.get('body').then(($body) => {
  if ($body.find('[data-cy="modal"]').length > 0) {
    // 如果模态框存在，关闭它
    cy.get('[data-cy="modal-close"]').click()
  }
})

// 链式处理多个异步操作
cy.fixture('users.json')
  .then((users) => {
    return users[0]  // 取第一个用户
  })
  .then((user) => {
    cy.get('[data-cy="name"]').type(user.name)
    cy.get('[data-cy="email"]').type(user.email)
  })

// wrap() 用于非 Cypress 对象
const data = { name: '张三', age: 30 }
cy.wrap(data)
  .should('have.property', 'name', '张三')
  .should('have.property', 'age', 30)
```

## 9. 如何测试表单和输入验证？

**答案：**
组合使用 `type()`、`select()`、`check()` 等命令和断言。

**代码示例：**
```javascript
describe('表单测试', () => {
  beforeEach(() => {
    cy.visit('/register')
  })

  it('应该验证必填字段', () => {
    // 不填写任何内容，直接提交
    cy.get('[data-cy="submit"]').click()

    // 验证错误提示
    cy.get('[data-cy="name-error"]')
      .should('be.visible')
      .should('contain', '姓名不能为空')

    cy.get('[data-cy="email-error"]')
      .should('be.visible')
      .should('contain', '邮箱不能为空')
  })

  it('应该验证邮箱格式', () => {
    cy.get('[data-cy="email"]').type('invalid-email')
    cy.get('[data-cy="email"]').blur()  // 失去焦点触发验证

    cy.get('[data-cy="email-error"]')
      .should('contain', '请输入有效的邮箱地址')
  })

  it('应该成功提交表单', () => {
    // 文本输入
    cy.get('[data-cy="name"]').type('张三')
    cy.get('[data-cy="email"]').type('zhangsan@example.com')

    // 下拉选择
    cy.get('[data-cy="country"]').select('中国')
    cy.get('[data-cy="country"]').should('have.value', 'CN')

    // 复选框
    cy.get('[data-cy="terms"]').check()
    cy.get('[data-cy="terms"]').should('be.checked')

    // 单选按钮
    cy.get('[data-cy="gender-male"]').check()
    cy.get('[data-cy="gender-male"]').should('be.checked')

    // 文本域
    cy.get('[data-cy="bio"]').type('这是我的个人简介')

    // 提交
    cy.get('[data-cy="submit"]').click()

    // 验证成功
    cy.url().should('include', '/success')
    cy.contains('注册成功').should('be.visible')
  })

  it('应该能够取消选择', () => {
    cy.get('[data-cy="newsletter"]').check().should('be.checked')
    cy.get('[data-cy="newsletter"]').uncheck().should('not.be.checked')
  })
})
```

## 10. 如何处理条件测试和动态内容？

**答案：**
使用 `.then()` 检查元素状态，根据条件执行不同操作。

**代码示例：**
```javascript
// 条件检查：元素是否存在
cy.get('body').then(($body) => {
  if ($body.find('[data-cy="cookie-banner"]').length > 0) {
    cy.get('[data-cy="accept-cookies"]').click()
  }
})

// 条件检查：元素状态
cy.get('[data-cy="toggle"]').then(($toggle) => {
  if ($toggle.hasClass('active')) {
    cy.log('开关已开启')
  } else {
    cy.wrap($toggle).click()
  }
})

// 条件检查：文本内容
cy.get('[data-cy="status"]')
  .invoke('text')
  .then((text) => {
    if (text.includes('离线')) {
      cy.get('[data-cy="connect-btn"]').click()
      cy.get('[data-cy="status"]').should('contain', '在线')
    }
  })

// 实际应用：处理随机 UI
cy.get('body').then(($body) => {
  // 检查是否有推广弹窗
  if ($body.find('[data-cy="promo-modal"]').length > 0) {
    cy.get('[data-cy="close-promo"]').click()
  }

  // 检查是否需要登录
  if ($body.find('[data-cy="login-form"]').length > 0) {
    cy.get('[data-cy="username"]').type('testuser')
    cy.get('[data-cy="password"]').type('pass123')
    cy.get('[data-cy="login-btn"]').click()
  }
})

// 重试机制：等待动态内容
cy.get('[data-cy="user-count"]')
  .should(($el) => {
    const count = parseInt($el.text())
    expect(count).to.be.greaterThan(0)
  })

// 多个可能的结果
cy.get('[data-cy="result"]')
  .should(($result) => {
    const text = $result.text()
    expect(['成功', '处理中', '已完成']).to.include(text)
  })
```
