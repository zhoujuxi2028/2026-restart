## 1. 什么是 Cypress？它与其他测试框架（如 Selenium）有什么区别？

**答案**:

Cypress 是一个现代化的前端端到端测试框架，直接在浏览器中运行。

**主要区别**:

| 特性 | Cypress | Selenium |
|-----|---------|----------|
| 架构 | 直接在浏览器中运行 | 通过 WebDriver 协议 |
| 执行速度 | 快速（无网络延迟） | 较慢（网络通信） |
| 调试能力 | 时间旅行、自动快照 | 基本日志 |
| 等待机制 | 自动重试和等待 | 需要显式等待 |
| 异步处理 | 自动处理 | 手动管理 Promise |

```javascript
// Cypress - 简洁直观
cy.get('.submit-button').click();
cy.url().should('include', '/dashboard');

// Selenium - 需要更多配置
await driver.findElement(By.css('.submit-button')).click();
await driver.wait(until.urlContains('/dashboard'), 5000);
```

**关键点**:
- Cypress 不使用 WebDriver
- 实时重载和时间旅行调试
- 自动等待元素可用
- 更适合现代 JavaScript 应用

## 2. Cypress 的核心架构是什么？为什么它执行速度快？

**答案**:

Cypress 采用独特的"浏览器内"架构，测试代码和应用代码运行在同一运行循环中。

**架构组件**:

```
┌─────────────────────────────────┐
│     Node.js 进程                │
│  - 文件系统访问                  │
│  - 网络代理                     │
│  - 任务执行                     │
└────────────┬────────────────────┘
             │
             ↓
┌─────────────────────────────────┐
│     浏览器进程                   │
│  - 测试代码                     │
│  - 应用代码                     │
│  - Cypress 运行时               │
└─────────────────────────────────┘
```

```javascript
// Cypress 可以直接访问应用对象
cy.window().then((win) => {
  // 直接访问应用的 window 对象
  expect(win.appState).to.exist;
});

// 可以直接操作 DOM
cy.document().then((doc) => {
  doc.querySelector('.element').style.color = 'red';
});
```

**关键点**:
- 无网络延迟，直接函数调用
- 完全控制网络层（可拦截请求）
- 实时访问应用状态
- 同步访问 DOM 和浏览器 API

## 3. Cypress 的命令队列机制是如何工作的？

**答案**:

Cypress 命令不会立即执行，而是被加入队列，按顺序异步执行。

```javascript
// ❌ 错误理解 - 这不是同步的
const button = cy.get('button');  // button 是 Chainable 对象，不是 DOM 元素
button.click();  // 会报错

// ✅ 正确方式 - 链式调用
cy.get('button').click();

// ✅ 使用 then() 获取实际值
cy.get('button').then(($btn) => {
  // $btn 是 jQuery 对象
  const text = $btn.text();
  cy.log(text);
});

// 命令执行顺序
cy.log('1 - 加入队列');
cy.get('.element').click();
cy.log('2 - 加入队列');
console.log('3 - 立即执行（同步）');
// 输出顺序: 3 -> 1 -> click -> 2
```

**命令队列示例**:

```javascript
cy.visit('/page')           // 队列: [visit]
  .get('.input')            // 队列: [visit, get]
  .type('hello')            // 队列: [visit, get, type]
  .should('have.value', 'hello');  // 队列: [visit, get, type, should]
```

**关键点**:
- 所有 `cy.*` 命令都是异步的
- 命令按添加顺序执行
- 使用 `.then()` 处理异步结果
- 不能直接将命令结果赋值给变量

## 4. 如何在 Cypress 中使用别名 (Aliases)？

**答案**:

别名用于保存和重用元素、请求或值。

```javascript
// 1. 元素别名
cy.get('.user-list').as('users');
cy.get('@users').should('have.length', 5);
cy.get('@users').first().click();

// 2. 请求别名
cy.intercept('GET', '/api/users').as('getUsers');
cy.visit('/users');
cy.wait('@getUsers').then((interception) => {
  expect(interception.response.statusCode).to.eq(200);
});

// 3. 值别名
cy.wrap({ name: 'John', age: 30 }).as('user');
cy.get('@user').its('name').should('eq', 'John');

// 4. 在钩子中使用别名
beforeEach(() => {
  cy.fixture('user.json').as('userData');
});

it('使用别名数据', function() {
  // 通过 this.userData 访问
  cy.log(this.userData.email);
});

// 5. 多次重用别名
cy.get('table').find('tr').as('rows');
cy.get('@rows').should('have.length', 10);
cy.get('@rows').first().should('contain', 'Header');
```

**关键点**:
- 使用 `.as()` 创建别名
- 使用 `@` 符号引用别名
- 在箭头函数中使用 `cy.get('@alias')`
- 在普通函数中使用 `this.alias`
- 别名会自动重新查询 DOM

## 5. Cypress 中的 fixtures 是什么？如何使用？

**答案**:

Fixtures 是存储在 `cypress/fixtures/` 目录中的静态测试数据文件。

```javascript
// cypress/fixtures/user.json
{
  "email": "test@example.com",
  "password": "password123",
  "profile": {
    "name": "Test User",
    "role": "admin"
  }
}

// 使用方式 1: 直接加载
cy.fixture('user.json').then((user) => {
  cy.get('#email').type(user.email);
  cy.get('#password').type(user.password);
});

// 使用方式 2: 结合别名
beforeEach(() => {
  cy.fixture('user').as('user');
});

it('登录测试', function() {
  cy.visit('/login');
  cy.get('#email').type(this.user.email);
  cy.get('#password').type(this.user.password);
});

// 使用方式 3: 模拟 API 响应
cy.intercept('GET', '/api/user', {
  fixture: 'user.json'
}).as('getUser');

// 使用方式 4: 支持多种格式
cy.fixture('data.csv');      // CSV
cy.fixture('image.png');     // 二进制文件
cy.fixture('config.json');   // JSON

// 使用方式 5: 动态修改
cy.fixture('user').then((user) => {
  user.email = 'newemail@example.com';
  cy.wrap(user).as('modifiedUser');
});
```

**关键点**:
- 默认在 `cypress/fixtures/` 目录
- 支持 JSON、JS、CSV、图片等格式
- 可与 `cy.intercept()` 结合模拟 API
- 使用别名可在测试间共享数据

## 6. 什么是 Cypress 的"时间旅行"功能？

**答案**:

时间旅行允许在测试执行后查看每个命令执行时的应用状态。

```javascript
describe('时间旅行示例', () => {
  it('展示时间旅行', () => {
    cy.visit('/todo');
    cy.get('.new-todo').type('Buy milk{enter}');  // 快照 1
    cy.get('.todo-list li').should('have.length', 1);  // 快照 2
    cy.get('.todo-list li').first().click();  // 快照 3
    cy.get('.completed').should('have.length', 1);  // 快照 4
  });
});

// 在 Cypress Test Runner 中：
// - 点击左侧命令可查看该时刻的 DOM
// - 鼠标悬停可查看命令详情
// - 可以在任意时刻查看应用状态
```

**功能特性**:

| 功能 | 说明 |
|-----|------|
| DOM 快照 | 每个命令执行前后的快照 |
| 控制台输出 | 记录所有日志和错误 |
| 网络请求 | 查看请求和响应 |
| 命令日志 | 详细的执行步骤 |

**调试示例**:

```javascript
cy.get('.item')
  .should('have.length', 5)  // 失败时点击查看实际 DOM
  .first()
  .click()  // 查看点击时的状态
  .should('have.class', 'active');  // 查看断言失败原因
```

**关键点**:
- 仅在交互模式下可用
- 自动保存每个命令的快照
- 无需额外配置
- 极大简化调试过程

## 7. Cypress 支持哪些浏览器？如何配置？

**答案**:

Cypress 支持 Chromium 系、Firefox 和 Edge 浏览器。

**支持的浏览器**:

```javascript
// cypress.config.js
const { defineConfig } = require('cypress');

module.exports = defineConfig({
  e2e: {
    // 默认浏览器
    browser: 'chrome',

    // 浏览器启动选项
    chromeWebSecurity: false,  // 禁用同源策略
  }
});
```

**命令行指定浏览器**:

```bash
# Chrome (默认)
npx cypress run --browser chrome

# Firefox
npx cypress run --browser firefox

# Edge
npx cypress run --browser edge

# Electron (无头模式)
npx cypress run --browser electron

# 指定浏览器路径
npx cypress run --browser /usr/bin/chromium
```

**在测试中检测浏览器**:

```javascript
it('跨浏览器测试', () => {
  if (Cypress.isBrowser('chrome')) {
    // Chrome 特定逻辑
    cy.log('Running in Chrome');
  }

  if (Cypress.isBrowser('firefox')) {
    // Firefox 特定逻辑
    cy.log('Running in Firefox');
  }

  // 检查浏览器系列
  if (Cypress.isBrowser(['chrome', 'edge'])) {
    // Chromium 系浏览器
  }
});

// 跳过特定浏览器
it('仅在 Chrome 运行', { browser: 'chrome' }, () => {
  // 测试代码
});
```

**关键点**:
- 不支持 Safari (使用 WebKit 引擎的 Electron)
- Electron 是默认的无头浏览器
- 可以针对不同浏览器编写条件逻辑
- 使用 `--headed` 可在有头模式运行

## 8. Cypress 测试的基本结构是什么？

**答案**:

Cypress 使用 Mocha 的 BDD 语法结构测试。

```javascript
// 测试套件
describe('用户登录功能', () => {

  // 前置条件 - 所有测试前执行一次
  before(() => {
    cy.log('测试套件开始');
    // 初始化数据库等
  });

  // 前置条件 - 每个测试前执行
  beforeEach(() => {
    cy.visit('/login');
    cy.clearCookies();
    cy.clearLocalStorage();
  });

  // 后置条件 - 每个测试后执行
  afterEach(() => {
    cy.screenshot();
  });

  // 后置条件 - 所有测试后执行一次
  after(() => {
    cy.log('测试套件结束');
  });

  // 测试用例
  it('应该成功登录', () => {
    cy.get('#email').type('user@example.com');
    cy.get('#password').type('password123');
    cy.get('button[type="submit"]').click();
    cy.url().should('include', '/dashboard');
  });

  // 仅运行此测试
  it.only('应该显示错误消息', () => {
    cy.get('button[type="submit"]').click();
    cy.get('.error').should('be.visible');
  });

  // 跳过此测试
  it.skip('密码重置功能', () => {
    // 暂时跳过
  });

  // 嵌套套件
  context('输入验证', () => {
    it('应该验证邮箱格式', () => {
      cy.get('#email').type('invalid-email');
      cy.get('.email-error').should('be.visible');
    });
  });
});
```

**执行钩子顺序**:

```javascript
describe('测试套件', () => {
  before(() => console.log('1. before'));
  beforeEach(() => console.log('2. beforeEach'));
  afterEach(() => console.log('4. afterEach'));
  after(() => console.log('5. after'));

  it('测试1', () => console.log('3. 测试1'));
  it('测试2', () => console.log('3. 测试2'));
});

// 输出:
// 1. before
// 2. beforeEach
// 3. 测试1
// 4. afterEach
// 2. beforeEach
// 3. 测试2
// 4. afterEach
// 5. after
```

**关键点**:
- `describe()` / `context()` 定义测试套件
- `it()` 定义测试用例
- 支持钩子函数进行设置和清理
- 可使用 `.only` 和 `.skip` 控制执行

## 9. 如何在 Cypress 中处理环境变量？

**答案**:

Cypress 支持多种方式配置和使用环境变量。

```javascript
// 方式 1: cypress.config.js
module.exports = defineConfig({
  e2e: {
    env: {
      apiUrl: 'https://api.example.com',
      username: 'testuser'
    }
  }
});

// 方式 2: cypress.env.json (不提交到 git)
{
  "apiUrl": "https://api.example.com",
  "password": "secret123",
  "apiKey": "abc123xyz"
}

// 方式 3: 命令行
// npx cypress run --env apiUrl=https://staging.api.com,username=test

// 方式 4: 系统环境变量
// export CYPRESS_API_URL=https://api.example.com

// 在测试中使用
describe('环境变量测试', () => {
  it('访问 API', () => {
    const apiUrl = Cypress.env('apiUrl');
    cy.request(`${apiUrl}/users`).then((response) => {
      expect(response.status).to.eq(200);
    });
  });

  it('使用凭证', () => {
    cy.visit('/login');
    cy.get('#username').type(Cypress.env('username'));
    cy.get('#password').type(Cypress.env('password'));
  });

  // 设置环境变量
  it('动态设置', () => {
    Cypress.env('token', 'new-token-123');
    expect(Cypress.env('token')).to.eq('new-token-123');
  });
});

// 不同环境配置
// cypress.staging.json
{
  "baseUrl": "https://staging.example.com",
  "env": {
    "apiUrl": "https://api.staging.example.com"
  }
}

// package.json
{
  "scripts": {
    "test:staging": "cypress run --config-file cypress.staging.json",
    "test:prod": "cypress run --config-file cypress.prod.json"
  }
}
```

**优先级（从高到低）**:

1. 命令行 `--env`
2. 系统环境变量 `CYPRESS_*`
3. `cypress.env.json`
4. `cypress.config.js` 中的 `env`
5. 插件文件设置

**关键点**:
- 敏感信息放在 `cypress.env.json` 并加入 `.gitignore`
- 使用 `Cypress.env()` 访问
- 支持动态设置和更新
- 系统变量需加 `CYPRESS_` 前缀

## 10. Cypress 中的自定义命令如何创建和使用？

**答案**:

自定义命令可以封装常用操作，提高代码复用性。

```javascript
// cypress/support/commands.js

// 1. 简单自定义命令
Cypress.Commands.add('login', (email, password) => {
  cy.visit('/login');
  cy.get('#email').type(email);
  cy.get('#password').type(password);
  cy.get('button[type="submit"]').click();
});

// 2. 带默认参数的命令
Cypress.Commands.add('login', (
  email = 'test@example.com',
  password = 'password123'
) => {
  cy.session([email, password], () => {
    cy.visit('/login');
    cy.get('#email').type(email);
    cy.get('#password').type(password);
    cy.get('button[type="submit"]').click();
    cy.url().should('include', '/dashboard');
  });
});

// 3. 子命令（链式调用）
Cypress.Commands.add('setText', { prevSubject: 'element' }, ($element, text) => {
  return cy.wrap($element).clear().type(text);
});

// 4. 双命令（可选链式调用）
Cypress.Commands.add('console', { prevSubject: 'optional' }, (subject, method) => {
  console[method](subject);
  return subject;
});

// 5. 覆盖现有命令
Cypress.Commands.overwrite('visit', (originalFn, url, options) => {
  const domain = Cypress.env('baseUrl');
  if (domain) {
    url = `${domain}${url}`;
  }
  return originalFn(url, options);
});

// 6. 带选项的命令
Cypress.Commands.add('createUser', (userData, options = {}) => {
  const defaults = {
    role: 'user',
    active: true
  };

  const user = { ...defaults, ...userData };

  return cy.request({
    method: 'POST',
    url: '/api/users',
    body: user,
    ...options
  });
});
```

**使用自定义命令**:

```javascript
// 测试文件中使用
describe('用户功能', () => {
  it('用户登录', () => {
    cy.login();  // 使用默认参数
    cy.url().should('include', '/dashboard');
  });

  it('管理员登录', () => {
    cy.login('admin@example.com', 'admin123');
  });

  it('使用子命令', () => {
    cy.get('#input').setText('新文本');
  });

  it('创建用户', () => {
    cy.createUser({
      email: 'newuser@example.com',
      name: 'New User'
    }).then((response) => {
      expect(response.status).to.eq(201);
    });
  });
});
```

**TypeScript 类型定义**:

```typescript
// cypress/support/index.d.ts
declare namespace Cypress {
  interface Chainable {
    login(email?: string, password?: string): Chainable<void>;
    setText(text: string): Chainable<Element>;
    createUser(userData: object, options?: object): Chainable<Response>;
  }
}
```

**关键点**:
- 在 `cypress/support/commands.js` 中定义
- 三种类型：普通、子命令、双命令
- 使用 `prevSubject` 控制链式调用
- 可以覆盖内置命令
- 提供 TypeScript 类型支持

## 11. Cypress 的测试重试机制是如何工作的？

**答案**:

Cypress 支持测试失败后自动重试，提高测试稳定性。

```javascript
// cypress.config.js - 全局配置
module.exports = defineConfig({
  e2e: {
    retries: {
      runMode: 2,      // cypress run 时重试 2 次
      openMode: 0      // cypress open 时不重试
    }
  }
});

// 测试级别配置
describe('用户功能', () => {
  // 套件级别
  describe('登录功能', {
    retries: {
      runMode: 3,
      openMode: 1
    }
  }, () => {
    it('应该成功登录', () => {
      cy.visit('/login');
      cy.get('#email').type('user@example.com');
      cy.get('#password').type('password123');
      cy.get('button').click();
    });
  });

  // 单个测试级别
  it('不稳定的测试', { retries: 2 }, () => {
    // 这个测试失败后会重试 2 次
    cy.visit('/flaky-page');
    cy.get('.dynamic-content').should('be.visible');
  });

  // 禁用重试
  it('关键测试不重试', { retries: 0 }, () => {
    cy.visit('/critical-page');
  });
});
```

**重试行为**:

```javascript
describe('重试示例', () => {
  let attempts = 0;

  beforeEach(() => {
    // beforeEach 会在每次重试时执行
    attempts++;
    cy.log(`尝试次数: ${attempts}`);
  });

  it('演示重试', { retries: 2 }, () => {
    // 第1次: 失败
    // 第2次: beforeEach + 测试 -> 失败
    // 第3次: beforeEach + 测试 -> 失败
    // 总共执行 3 次（1次初始 + 2次重试）
    cy.wrap(attempts).should('be.gt', 2);
  });
});
```

**重试策略**:

```javascript
// 仅在特定错误时重试
it('智能重试', { retries: 2 }, () => {
  cy.on('fail', (error) => {
    // 网络错误才重试
    if (error.message.includes('Network')) {
      throw error;  // 触发重试
    } else {
      throw new Error('不重试: ' + error.message);
    }
  });

  cy.request('/api/data');
});

// 使用环境变量控制
const retries = Cypress.env('ci') ? 3 : 0;

it('CI 环境重试', { retries }, () => {
  // CI 环境重试 3 次，本地不重试
});
```

**关键点**:
- 区分 `runMode`（CI）和 `openMode`（本地调试）
- 重试会重新执行 `beforeEach` 钩子
- 配置优先级: 测试级 > 套件级 > 全局
- 避免过度依赖重试，应修复不稳定测试

## 12. 如何在 Cypress 中处理 iframe？

**答案**:

Cypress 需要特殊方法来访问 iframe 内的元素。

```javascript
// 方法 1: 自定义命令（推荐）
Cypress.Commands.add('getIframe', (iframeSelector) => {
  return cy.get(iframeSelector)
    .its('0.contentDocument.body')
    .should('not.be.empty')
    .then(cy.wrap);
});

// 使用
cy.getIframe('#my-iframe')
  .find('.button')
  .click();

// 方法 2: 完整的 iframe 处理
Cypress.Commands.add('iframe', (iframeSelector) => {
  return cy.get(iframeSelector)
    .its('0.contentDocument')
    .should('exist')
    .its('body')
    .should('not.be.undefined')
    .then(cy.wrap);
});

// 方法 3: 等待 iframe 加载
cy.get('iframe#payment')
  .should('be.visible')
  .then(($iframe) => {
    const $body = $iframe.contents().find('body');
    cy.wrap($body)
      .find('#card-number')
      .type('4242424242424424');
  });

// 方法 4: 处理多层 iframe
Cypress.Commands.add('getNestedIframe', (outerSelector, innerSelector) => {
  return cy.get(outerSelector)
    .its('0.contentDocument.body')
    .should('not.be.empty')
    .then(cy.wrap)
    .find(innerSelector)
    .its('0.contentDocument.body')
    .should('not.be.empty')
    .then(cy.wrap);
});

// 实际应用示例
describe('iframe 测试', () => {
  it('支付表单测试', () => {
    cy.visit('/checkout');

    // 等待 iframe 加载
    cy.get('iframe[name="payment-frame"]')
      .should('be.visible')
      .and(($iframe) => {
        expect($iframe.contents().find('body')).to.exist;
      });

    // 操作 iframe 内容
    cy.iframe('iframe[name="payment-frame"]')
      .find('#card-number')
      .type('4242424242424424')
      .should('have.value', '4242424242424424');

    cy.iframe('iframe[name="payment-frame"]')
      .find('#expiry')
      .type('12/25');

    cy.iframe('iframe[name="payment-frame"]')
      .find('button[type="submit"]')
      .click();
  });
});
```

**注意事项**:

```javascript
// ❌ 不能直接访问
cy.get('iframe').find('.element');  // 不工作

// ✅ 需要先获取 contentDocument
cy.get('iframe')
  .its('0.contentDocument.body')
  .find('.element');

// 处理跨域 iframe（需要配置）
// cypress.config.js
{
  chromeWebSecurity: false  // 禁用安全策略
}
```

**关键点**:
- 需要通过 `contentDocument` 访问 iframe 内容
- 等待 iframe 完全加载
- 使用自定义命令简化操作
- 跨域 iframe 需禁用 web 安全
- 考虑使用第三方插件 `cypress-iframe`

## 14. Cypress 中的 cy.session() 是什么？如何提高测试速度？

**答案**:

`cy.session()` 用于缓存和恢复浏览器会话状态，避免重复登录。

```javascript
// 传统登录（每次测试都执行）
beforeEach(() => {
  cy.visit('/login');
  cy.get('#email').type('user@example.com');
  cy.get('#password').type('password123');
  cy.get('button').click();
  cy.url().should('include', '/dashboard');
});

// 使用 cy.session() 优化
beforeEach(() => {
  cy.session('user-session', () => {
    // 这里的代码只在第一次执行
    cy.visit('/login');
    cy.get('#email').type('user@example.com');
    cy.get('#password').type('password123');
    cy.get('button').click();
    cy.url().should('include', '/dashboard');
  }, {
    validate() {
      // 验证会话是否仍然有效
      cy.getCookie('session-id').should('exist');
    },
    cacheAcrossSpecs: true  // 跨测试文件缓存
  });

  // 会话恢复后，直接访问需要登录的页面
  cy.visit('/dashboard');
});

// 高级用法：多用户会话
const login = (username, password) => {
  cy.session([username, password], () => {
    cy.visit('/login');
    cy.get('#email').type(username);
    cy.get('#password').type(password);
    cy.get('button').click();
  }, {
    validate() {
      cy.request('/api/user').its('status').should('eq', 200);
    }
  });
};

describe('用户功能', () => {
  it('普通用户测试', () => {
    login('user@example.com', 'password123');
    cy.visit('/profile');
  });

  it('管理员测试', () => {
    login('admin@example.com', 'admin123');
    cy.visit('/admin');
  });
});

// 自定义命令封装
Cypress.Commands.add('loginByAPI', (email, password) => {
  cy.session([email, password], () => {
    cy.request({
      method: 'POST',
      url: '/api/login',
      body: { email, password }
    }).then((response) => {
      window.localStorage.setItem('token', response.body.token);
    });
  }, {
    validate() {
      const token = window.localStorage.getItem('token');
      expect(token).to.exist;
    }
  });
});

// 使用
it('API 登录测试', () => {
  cy.loginByAPI('user@example.com', 'password123');
  cy.visit('/dashboard');
  cy.get('.user-name').should('contain', 'User');
});
```

**性能对比**:

```javascript
// 没有 cy.session()
// 10 个测试 = 10 次完整登录 = ~30 秒

// 使用 cy.session()
// 10 个测试 = 1 次登录 + 9 次会话恢复 = ~8 秒
```

**关键点**:
- 缓存 cookies、localStorage、sessionStorage
- 使用唯一 ID 标识会话
- 提供 `validate()` 验证会话有效性
- 跨测试文件缓存可选
- 显著提高测试套件速度

## 15. Cypress 的插件系统是如何工作的？

**答案**:

Cypress 插件在 Node.js 进程中运行，提供扩展能力。

```javascript
// cypress.config.js
const { defineConfig } = require('cypress');

module.exports = defineConfig({
  e2e: {
    setupNodeEvents(on, config) {
      // 1. 任务注册
      on('task', {
        // 数据库操作
        'db:seed'() {
          // 连接数据库并插入测试数据
          return require('./db').seed();
        },

        // 读取文件
        'readFile'(filename) {
          return require('fs').readFileSync(filename, 'utf8');
        },

        // 日志记录
        log(message) {
          console.log(message);
          return null;
        }
      });

      // 2. 浏览器启动配置
      on('before:browser:launch', (browser, launchOptions) => {
        if (browser.name === 'chrome') {
          launchOptions.args.push('--disable-dev-shm-usage');
        }
        return launchOptions;
      });

      // 3. 文件预处理器（TypeScript）
      on('file:preprocessor', require('@cypress/webpack-preprocessor'));

      // 4. 截图处理
      on('after:screenshot', (details) => {
        console.log('截图保存:', details.path);
        return details;
      });

      // 5. 配置修改
      config.baseUrl = process.env.BASE_URL || config.baseUrl;
      config.env.apiUrl = process.env.API_URL || config.env.apiUrl;

      return config;
    }
  }
});

// 在测试中使用任务
describe('插件测试', () => {
  before(() => {
    // 调用数据库种子任务
    cy.task('db:seed');
  });

  it('使用自定义任务', () => {
    cy.task('log', '开始测试');
    cy.visit('/');
    cy.task('readFile', 'data.json').then((content) => {
      cy.log(content);
    });
  });
});
```

**常用插件示例**:

```javascript
// 1. 代码覆盖率插件
// cypress.config.js
const { defineConfig } = require('cypress');
const codeCoverageTask = require('@cypress/code-coverage/task');

module.exports = defineConfig({
  e2e: {
    setupNodeEvents(on, config) {
      codeCoverageTask(on, config);
      return config;
    }
  }
});

// 2. 自定义插件创建
// cypress/plugins/custom-plugin.js
module.exports = (on, config) => {
  on('task', {
    async 'api:createUser'(userData) {
      const axios = require('axios');
      const response = await axios.post('/api/users', userData);
      return response.data;
    }
  });

  on('before:run', (details) => {
    console.log('测试开始运行:', details.specs.length, '个文件');
  });

  on('after:run', (results) => {
    console.log('测试完成，失败:', results.totalFailed);
  });

  return config;
};

// 使用
cy.task('api:createUser', {
  name: 'John Doe',
  email: 'john@example.com'
});
```

**事件监听器**:

| 事件 | 触发时机 | 用途 |
|-----|---------|------|
| `before:browser:launch` | 浏览器启动前 | 配置启动参数 |
| `before:run` | 测试运行前 | 初始化设置 |
| `after:run` | 测试运行后 | 清理、报告 |
| `after:screenshot` | 截图后 | 处理截图文件 |
| `file:preprocessor` | 文件加载前 | 编译 TS/JSX |
| `task` | 测试中调用 | 执行 Node 任务 |

**关键点**:
- 插件在 Node.js 环境运行
- 可访问文件系统、数据库等
- 使用 `cy.task()` 在测试中调用
- 支持异步操作
- 不能直接访问浏览器或 DOM
