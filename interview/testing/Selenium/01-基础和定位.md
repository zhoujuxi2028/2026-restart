## 1. Selenium 是什么？它的核心组件有哪些？

**答案：**
Selenium 是一个开源的 Web 自动化测试框架，用于模拟用户在浏览器中的操作。

核心组件：
- **Selenium WebDriver**：核心API，直接与浏览器通信
- **Selenium Grid**：分布式测试，支持并行执行
- **Selenium IDE**：浏览器插件，用于录制和回放

架构原理：
1. 测试脚本通过 WebDriver API 发送命令
2. 浏览器驱动（ChromeDriver/GeckoDriver）接收命令
3. 驱动通过浏览器协议控制浏览器
4. 浏览器执行操作并返回结果

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

# WebDriver 是核心接口
driver = webdriver.Chrome()
driver.get("https://www.example.com")

# 通过 WebDriver 控制浏览器
element = driver.find_element(By.ID, "username")
element.send_keys("testuser")

driver.quit()
```

## 2. 如何初始化不同浏览器的 WebDriver？说明常用配置选项。

**答案：**
每种浏览器需要对应的驱动程序，可以手动下载或使用 webdriver-manager 自动管理。

```python
from selenium import webdriver
from selenium.webdriver.chrome.options import Options as ChromeOptions
from selenium.webdriver.firefox.options import Options as FirefoxOptions
from selenium.webdriver.edge.options import Options as EdgeOptions

# 1. Chrome 浏览器
chrome_options = ChromeOptions()
chrome_options.add_argument('--headless')  # 无头模式
chrome_options.add_argument('--disable-gpu')
chrome_options.add_argument('--no-sandbox')
chrome_options.add_argument('--window-size=1920,1080')
chrome_options.add_experimental_option('excludeSwitches', ['enable-logging'])
driver = webdriver.Chrome(options=chrome_options)

# 2. Firefox 浏览器
firefox_options = FirefoxOptions()
firefox_options.add_argument('--headless')
firefox_options.set_preference('browser.download.folderList', 2)
driver = webdriver.Firefox(options=firefox_options)

# 3. Edge 浏览器
edge_options = EdgeOptions()
edge_options.use_chromium = True
driver = webdriver.Edge(options=edge_options)

# 使用 webdriver-manager 自动管理驱动
from webdriver_manager.chrome import ChromeDriverManager
from selenium.webdriver.chrome.service import Service

service = Service(ChromeDriverManager().install())
driver = webdriver.Chrome(service=service, options=chrome_options)
```

## 3. Selenium 支持哪8种元素定位方法？各自适用场景是什么？

**答案：**
Selenium 4.x 推荐使用 `By` 类进行定位，共8种方法。

| 定位方法 | 语法 | 适用场景 | 优先级 |
|---------|------|---------|-------|
| ID | `By.ID` | 元素有唯一ID | ⭐⭐⭐⭐⭐ |
| NAME | `By.NAME` | 表单元素 | ⭐⭐⭐⭐ |
| CLASS_NAME | `By.CLASS_NAME` | 单个类名 | ⭐⭐⭐ |
| TAG_NAME | `By.TAG_NAME` | 查找所有同类标签 | ⭐⭐ |
| LINK_TEXT | `By.LINK_TEXT` | 完整链接文本 | ⭐⭐⭐⭐ |
| PARTIAL_LINK_TEXT | `By.PARTIAL_LINK_TEXT` | 部分链接文本 | ⭐⭐⭐ |
| CSS_SELECTOR | `By.CSS_SELECTOR` | 复杂定位，性能好 | ⭐⭐⭐⭐⭐ |
| XPATH | `By.XPATH` | 复杂关系，功能强 | ⭐⭐⭐⭐ |

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
driver.get("https://www.example.com")

# 1. ID 定位（最优先）
element = driver.find_element(By.ID, "username")

# 2. NAME 定位
element = driver.find_element(By.NAME, "email")

# 3. CLASS_NAME 定位（只能单个类名）
element = driver.find_element(By.CLASS_NAME, "btn-primary")

# 4. TAG_NAME 定位
elements = driver.find_elements(By.TAG_NAME, "input")

# 5. LINK_TEXT 定位（精确匹配）
element = driver.find_element(By.LINK_TEXT, "登录")

# 6. PARTIAL_LINK_TEXT 定位（部分匹配）
element = driver.find_element(By.PARTIAL_LINK_TEXT, "注册")

# 7. CSS_SELECTOR 定位（推荐）
element = driver.find_element(By.CSS_SELECTOR, "input[type='password']")

# 8. XPATH 定位（功能最强）
element = driver.find_element(By.XPATH, "//input[@id='username']")

driver.quit()
```

## 4. CSS Selector 和 XPath 的区别？如何选择？

**答案：**

| 对比维度 | CSS Selector | XPath |
|---------|-------------|-------|
| 性能 | 更快（原生支持） | 较慢 |
| 语法 | 简洁 | 相对复杂 |
| 功能 | 有限制 | 功能强大 |
| 方向 | 只能向下查找 | 可以向上/向下查找 |
| 文本定位 | 不支持 | 支持 |
| 浏览器支持 | 所有现代浏览器 | 所有浏览器 |

**选择建议：**
- 优先使用 CSS Selector（性能好、语法简洁）
- 需要父节点查找或文本定位时使用 XPath

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
driver.get("https://www.example.com")

# CSS Selector 示例
# 1. ID选择器
element = driver.find_element(By.CSS_SELECTOR, "#username")

# 2. Class选择器
element = driver.find_element(By.CSS_SELECTOR, ".btn-primary")

# 3. 属性选择器
element = driver.find_element(By.CSS_SELECTOR, "input[name='email']")

# 4. 子元素选择器
element = driver.find_element(By.CSS_SELECTOR, "div.form-group > input")

# 5. 后代选择器
element = driver.find_element(By.CSS_SELECTOR, "form input[type='password']")

# 6. nth-child
element = driver.find_element(By.CSS_SELECTOR, "ul > li:nth-child(2)")

# XPath 示例
# 1. 绝对路径（不推荐）
element = driver.find_element(By.XPATH, "/html/body/div/form/input")

# 2. 相对路径（推荐）
element = driver.find_element(By.XPATH, "//input[@id='username']")

# 3. 文本定位
element = driver.find_element(By.XPATH, "//button[text()='登录']")

# 4. 包含文本
element = driver.find_element(By.XPATH, "//a[contains(text(), '注册')]")

# 5. 父节点查找（XPath独有）
element = driver.find_element(By.XPATH, "//input[@id='username']/parent::div")

# 6. 兄弟节点
element = driver.find_element(By.XPATH, "//label[@for='username']/following-sibling::input")

# 7. 逻辑运算
element = driver.find_element(By.XPATH, "//input[@type='text' and @name='username']")

driver.quit()
```

## 5. 如何定位动态 ID 或动态属性的元素？

**答案：**
动态属性指每次刷新页面后，元素的 ID 或 class 会变化（如 `user_12345`）。解决方法：

1. **使用部分匹配**（starts-with、contains、ends-with）
2. **使用稳定属性**（data-testid、aria-label）
3. **使用相对定位**（通过父元素或兄弟元素）
4. **使用文本定位**

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
driver.get("https://www.example.com")

# 场景：元素 ID 为 user_12345，每次刷新数字部分会变化

# 方法1：XPath - starts-with（ID以固定前缀开始）
element = driver.find_element(By.XPATH, "//div[starts-with(@id, 'user_')]")

# 方法2：XPath - contains（ID包含固定部分）
element = driver.find_element(By.XPATH, "//div[contains(@id, 'user')]")

# 方法3：CSS - 属性包含
element = driver.find_element(By.CSS_SELECTOR, "div[id*='user']")

# 方法4：CSS - 属性开头
element = driver.find_element(By.CSS_SELECTOR, "div[id^='user_']")

# 方法5：CSS - 属性结尾
element = driver.find_element(By.CSS_SELECTOR, "div[id$='_profile']")

# 方法6：使用稳定的 data 属性
element = driver.find_element(By.CSS_SELECTOR, "[data-testid='user-profile']")

# 方法7：通过父元素定位
element = driver.find_element(By.XPATH, "//div[@class='user-container']//div[contains(@id, 'user')]")

# 方法8：通过文本内容定位
element = driver.find_element(By.XPATH, "//div[contains(text(), '用户信息')]")

# 方法9：组合多个稳定属性
element = driver.find_element(By.XPATH, "//input[@type='text' and @placeholder='请输入用户名']")

driver.quit()
```

## 6. find_element 和 find_elements 的区别？如何处理元素不存在的情况？

**答案：**

| 方法 | 返回值 | 找不到元素时 | 使用场景 |
|-----|--------|------------|---------|
| find_element | 单个 WebElement | 抛出 NoSuchElementException | 定位唯一元素 |
| find_elements | WebElement 列表 | 返回空列表 [] | 定位多个元素或判断存在性 |

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.common.exceptions import NoSuchElementException

driver = webdriver.Chrome()
driver.get("https://www.example.com")

# find_element - 单个元素
try:
    element = driver.find_element(By.ID, "username")
    element.send_keys("testuser")
except NoSuchElementException:
    print("元素不存在")

# find_elements - 多个元素
elements = driver.find_elements(By.CLASS_NAME, "product-item")
print(f"找到 {len(elements)} 个商品")

for index, element in enumerate(elements, 1):
    print(f"商品 {index}: {element.text}")

# 技巧1：使用 find_elements 判断元素是否存在（不抛异常）
def is_element_present(locator_type, locator_value):
    elements = driver.find_elements(locator_type, locator_value)
    return len(elements) > 0

if is_element_present(By.ID, "logout-button"):
    print("用户已登录")
else:
    print("用户未登录")

# 技巧2：安全获取元素
def safe_find_element(locator_type, locator_value, default=None):
    try:
        return driver.find_element(locator_type, locator_value)
    except NoSuchElementException:
        return default

element = safe_find_element(By.ID, "optional-field")
if element:
    element.send_keys("optional data")

# 技巧3：获取第一个元素（如果存在）
elements = driver.find_elements(By.CLASS_NAME, "error-message")
if elements:
    first_error = elements[0]
    print(f"错误信息: {first_error.text}")

driver.quit()
```

## 7. 如何定位表格中的特定单元格？

**答案：**
表格定位需要结合行列索引或单元格内容进行定位。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
driver.get("https://www.example.com/table")

# 场景：定位表格中的特定数据
# HTML结构：
# <table id="data-table">
#   <thead><tr><th>姓名</th><th>年龄</th><th>操作</th></tr></thead>
#   <tbody>
#     <tr><td>张三</td><td>25</td><td><button>编辑</button></td></tr>
#     <tr><td>李四</td><td>30</td><td><button>编辑</button></td></tr>
#   </tbody>
# </table>

# 方法1：通过行列索引定位（第2行第3列）
cell = driver.find_element(By.XPATH, "//table[@id='data-table']//tr[2]/td[3]")

# 方法2：定位所有行
rows = driver.find_elements(By.XPATH, "//table[@id='data-table']//tbody/tr")
print(f"表格共有 {len(rows)} 行")

# 方法3：遍历表格数据
for row in rows:
    cells = row.find_elements(By.TAG_NAME, "td")
    name = cells[0].text
    age = cells[1].text
    print(f"姓名: {name}, 年龄: {age}")

# 方法4：根据单元格内容定位行
def find_row_by_name(name):
    return driver.find_element(By.XPATH, f"//table[@id='data-table']//td[text()='{name}']/parent::tr")

zhang_san_row = find_row_by_name("张三")

# 方法5：定位特定行的操作按钮
edit_button = driver.find_element(
    By.XPATH,
    "//table[@id='data-table']//td[text()='张三']/following-sibling::td//button[text()='编辑']"
)
edit_button.click()

# 方法6：获取表头
headers = driver.find_elements(By.XPATH, "//table[@id='data-table']//thead//th")
header_texts = [header.text for header in headers]
print(f"表头: {header_texts}")

# 方法7：根据列名获取列索引
def get_column_index(header_name):
    headers = driver.find_elements(By.XPATH, "//table[@id='data-table']//thead//th")
    for index, header in enumerate(headers, 1):
        if header.text == header_name:
            return index
    return -1

age_column_index = get_column_index("年龄")
# 获取所有年龄数据
ages = driver.find_elements(By.XPATH, f"//table[@id='data-table']//tbody/tr/td[{age_column_index}]")

driver.quit()
```

## 8. 如何处理 shadow DOM 元素？

**答案：**
Shadow DOM 是 Web Components 的一部分，元素被封装在 shadow root 中，常规定位方法无法直接访问。需要先获取 shadow root，再定位内部元素。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
driver.get("https://www.example.com")

# 场景：元素在 shadow DOM 内部
# HTML结构：
# <div id="host">
#   #shadow-root
#     <input id="shadow-input" />
# </div>

# 方法1：使用 shadow_root 属性（Selenium 4+）
host_element = driver.find_element(By.ID, "host")
shadow_root = host_element.shadow_root
shadow_input = shadow_root.find_element(By.ID, "shadow-input")
shadow_input.send_keys("test")

# 方法2：使用 JavaScript 访问（兼容性更好）
host_element = driver.find_element(By.ID, "host")
shadow_input = driver.execute_script(
    "return arguments[0].shadowRoot.querySelector('#shadow-input')",
    host_element
)
shadow_input.send_keys("test")

# 方法3：多层嵌套的 shadow DOM
# <div id="host1">
#   #shadow-root
#     <div id="host2">
#       #shadow-root
#         <button id="nested-button"></button>

host1 = driver.find_element(By.ID, "host1")
shadow_root1 = host1.shadow_root
host2 = shadow_root1.find_element(By.ID, "host2")
shadow_root2 = host2.shadow_root
nested_button = shadow_root2.find_element(By.ID, "nested-button")
nested_button.click()

# 方法4：使用 JS 一次性定位深层 shadow DOM
nested_button = driver.execute_script("""
    return document.querySelector('#host1')
        .shadowRoot.querySelector('#host2')
        .shadowRoot.querySelector('#nested-button');
""")
nested_button.click()

# 封装工具函数
def find_element_in_shadow(driver, host_selector, shadow_selector):
    """在 shadow DOM 中定位元素"""
    return driver.execute_script(f"""
        return document.querySelector('{host_selector}')
            .shadowRoot.querySelector('{shadow_selector}');
    """)

element = find_element_in_shadow(driver, "#host", "#shadow-input")

driver.quit()
```

## 9. 如何使用相对定位（Relative Locators）？

**答案：**
Selenium 4 引入的相对定位功能，可以通过元素之间的空间关系定位（above、below、toLeftOf、toRightOf、near）。

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.relative_locator import locate_with

driver = webdriver.Chrome()
driver.get("https://www.example.com/form")

# 场景：表单中标签和输入框没有明确的关联属性
# HTML结构：
# <div>
#   <label>用户名</label>
#   <input id="username" />
# </div>
# <div>
#   <label>密码</label>
#   <input id="password" />
# </div>

# 方法1：above - 定位在某元素上方的元素
password_field = driver.find_element(By.ID, "password")
username_field = driver.find_element(locate_with(By.TAG_NAME, "input").above(password_field))
username_field.send_keys("testuser")

# 方法2：below - 定位在某元素下方的元素
username_field = driver.find_element(By.ID, "username")
password_field = driver.find_element(locate_with(By.TAG_NAME, "input").below(username_field))
password_field.send_keys("password123")

# 方法3：toLeftOf - 定位在某元素左侧的元素
submit_button = driver.find_element(By.ID, "submit")
cancel_button = driver.find_element(locate_with(By.TAG_NAME, "button").to_left_of(submit_button))

# 方法4：toRightOf - 定位在某元素右侧的元素
cancel_button = driver.find_element(By.ID, "cancel")
submit_button = driver.find_element(locate_with(By.TAG_NAME, "button").to_right_of(cancel_button))

# 方法5：near - 定位在某元素附近的元素（50像素内）
label = driver.find_element(By.XPATH, "//label[text()='用户名']")
username_field = driver.find_element(locate_with(By.TAG_NAME, "input").near(label))

# 方法6：组合多个相对定位
# 定位在用户名字段下方且在密码标签右侧的输入框
username_field = driver.find_element(By.ID, "username")
password_label = driver.find_element(By.XPATH, "//label[text()='密码']")
password_field = driver.find_element(
    locate_with(By.TAG_NAME, "input")
    .below(username_field)
    .to_right_of(password_label)
)

# 实际应用：定位列表中某项旁边的操作按钮
product_name = driver.find_element(By.XPATH, "//span[text()='iPhone 15']")
delete_button = driver.find_element(
    locate_with(By.CLASS_NAME, "delete-btn").to_right_of(product_name)
)
delete_button.click()

driver.quit()
```

## 10. 元素定位的最佳实践是什么？

**答案：**
定位策略的优先级和最佳实践。

**定位策略优先级：**
1. ID（最优先，如果稳定）
2. data-testid 或 data-* 属性（测试专用属性）
3. NAME（表单元素）
4. CSS Selector（性能好、语法简洁）
5. XPath（需要复杂关系定位时）
6. LINK_TEXT/PARTIAL_LINK_TEXT（链接元素）
7. CLASS_NAME（避免使用动态类名）
8. TAG_NAME（最后选择）

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Chrome()

# 最佳实践1：优先使用 data-testid 属性（推荐前端添加）
# <button data-testid="login-submit">登录</button>
element = driver.find_element(By.CSS_SELECTOR, "[data-testid='login-submit']")

# 最佳实践2：避免使用绝对 XPath
# ❌ 不推荐
element = driver.find_element(By.XPATH, "/html/body/div[1]/div[2]/form/input[1]")
# ✅ 推荐
element = driver.find_element(By.XPATH, "//input[@name='username']")

# 最佳实践3：避免使用 index 定位（脆弱）
# ❌ 不推荐
element = driver.find_element(By.XPATH, "(//button)[3]")
# ✅ 推荐
element = driver.find_element(By.XPATH, "//button[text()='提交']")

# 最佳实践4：封装定位器（便于维护）
class LoginPageLocators:
    USERNAME = (By.ID, "username")
    PASSWORD = (By.ID, "password")
    SUBMIT = (By.CSS_SELECTOR, "[data-testid='login-submit']")
    ERROR_MSG = (By.CLASS_NAME, "error-message")

# 使用封装的定位器
driver.find_element(*LoginPageLocators.USERNAME).send_keys("testuser")
driver.find_element(*LoginPageLocators.PASSWORD).send_keys("password123")
driver.find_element(*LoginPageLocators.SUBMIT).click()

# 最佳实践5：结合显式等待
wait = WebDriverWait(driver, 10)
element = wait.until(EC.presence_of_element_located(LoginPageLocators.USERNAME))

# 最佳实践6：使用多个属性增加定位稳定性
element = driver.find_element(
    By.XPATH,
    "//input[@type='text' and @name='username' and @placeholder='请输入用户名']"
)

# 最佳实践7：创建可复用的定位函数
def find_button_by_text(driver, button_text):
    """通过按钮文本定位"""
    return driver.find_element(By.XPATH, f"//button[text()='{button_text}']")

def find_input_by_label(driver, label_text):
    """通过标签文本定位输入框"""
    return driver.find_element(
        By.XPATH,
        f"//label[text()='{label_text}']/following-sibling::input"
    )

submit_btn = find_button_by_text(driver, "提交")
username_input = find_input_by_label(driver, "用户名")

# 最佳实践8：处理动态属性
# ❌ 不推荐
element = driver.find_element(By.ID, "user_12345")  # ID动态变化
# ✅ 推荐
element = driver.find_element(By.XPATH, "//div[starts-with(@id, 'user_')]")

# 最佳实践9：避免依赖样式类名
# ❌ 不推荐
element = driver.find_element(By.CLASS_NAME, "MuiButton-root-123")  # 动态生成的类名
# ✅ 推荐
element = driver.find_element(By.CSS_SELECTOR, "button[aria-label='提交']")

# 最佳实践10：使用语义化属性
# 优先使用 aria-label, role, title 等语义化属性
element = driver.find_element(By.CSS_SELECTOR, "button[aria-label='关闭对话框']")
element = driver.find_element(By.CSS_SELECTOR, "[role='navigation']")

driver.quit()
```

**定位策略选择流程图：**
```
1. 元素有唯一稳定的 ID？ → 使用 ID
   ↓ 否
2. 前端有 data-testid？ → 使用 data-testid
   ↓ 否
```
