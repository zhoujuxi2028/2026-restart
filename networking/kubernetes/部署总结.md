## ⏳ 待完成的工作

### 关键步骤：启用 Kubernetes 集群

由于网络限制（无法从 Docker Hub 拉取镜像），推荐使用 **Docker Desktop 内置 Kubernetes**。

#### 启用步骤（2-3分钟）：

1. **打开 Docker Desktop 应用**
   - 确保 Docker Desktop 正在运行（菜单栏应该能看到 Docker 图标）

2. **进入设置**
   - 点击 Docker Desktop 右上角的 ⚙️ 设置图标

3. **启用 Kubernetes**
   - 左侧菜单选择 "Kubernetes"
   - 勾选 ☑️ "Enable Kubernetes"
   - 点击 "Apply & Restart" 按钮

4. **等待启动完成**
   - Docker Desktop 会下载并启动 Kubernetes 组件
   - 等待 2-3 分钟
   - 当 Docker Desktop 底部状态栏显示 "Kubernetes is running" 时表示成功

5. **验证安装**
   ```bash
   # 运行环境检查脚本
   cd /Users/michaelzhou/Documents/github/2026-restart/networking/kubernetes
   bash configs/check-environment.sh

   # 或手动验证
   kubectl cluster-info
   kubectl get nodes
   ```

## 🔧 故障排查

### 问题1：Minikube/Kind 无法启动
**原因**: 网络问题，无法从 Docker Hub 拉取镜像

**解决方案**: 使用 Docker Desktop 内置 K8S（不需要拉取镜像）

### 问题2：kubectl 命令无响应
**排查步骤**:
```bash
# 1. 检查集群是否运行
docker ps | grep k8s

# 2. 检查 kubeconfig
kubectl config current-context

# 3. 查看详细错误
kubectl cluster-info dump
```

### 问题3：Pod 一直 Pending
**常见原因**:
- 资源不足
- 镜像拉取失败
- 节点不可用

**排查命令**:
```bash
kubectl describe pod <pod-name>
kubectl get events
```
