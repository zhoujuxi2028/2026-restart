# 抓包分析实践教程
> 网络问题定位与性能诊断必备技能

## 🔧 工具介绍与环境准备

### Wireshark图形界面
**安装与配置**：
```bash
# macOS
brew install wireshark

# Ubuntu/Debian
sudo apt-get install wireshark

# 权限配置(Linux)
sudo usermod -a -G wireshark $USER
```

### tcpdump命令行工具
**基础语法**：
```bash
tcpdump [选项] [过滤表达式]
常用选项：
-i <interface>  指定网络接口
-w <file>       写入文件
-r <file>       读取文件
-n              不进行DNS解析
-v/-vv/-vvv     详细输出级别
-c <count>      捕获指定数量的包
```

## 📊 基础抓包操作

### 1. 网络接口识别
```bash
# 查看可用网络接口
ip addr show
ifconfig -a
wireshark -D  # Wireshark查看接口

# 选择合适的接口进行抓包
tcpdump -i eth0  # 以太网接口
tcpdump -i any   # 所有接口
tcpdump -i lo    # 回环接口
```

### 2. 基础过滤规则
```bash
# 按协议过滤
tcpdump -i eth0 icmp
tcpdump -i eth0 tcp
tcpdump -i eth0 udp

# 按主机过滤
tcpdump -i eth0 host 192.168.1.100
tcpdump -i eth0 src host 192.168.1.100
tcpdump -i eth0 dst host 192.168.1.100

# 按端口过滤
tcpdump -i eth0 port 80
tcpdump -i eth0 src port 80
tcpdump -i eth0 portrange 80-90

# 组合过滤
tcpdump -i eth0 host 192.168.1.100 and port 80
tcpdump -i eth0 '(host 192.168.1.100 or host 192.168.1.101) and port 80'
```

## 🔍 Wireshark实战分析

### 显示过滤器(Display Filters)
```bash
# 基础协议过滤
http
tcp
udp
dns
arp
icmp

# IP地址过滤
ip.addr == 192.168.1.100
ip.src == 192.168.1.100
ip.dst == 192.168.1.100

# 端口过滤
tcp.port == 80
tcp.srcport == 80
tcp.dstport == 80

# HTTP特定过滤
http.request.method == "GET"
http.response.code == 200
http.request.uri contains "api"

# TCP连接分析
tcp.flags.syn == 1
tcp.flags.fin == 1
tcp.flags.reset == 1

# 组合条件
(ip.addr == 192.168.1.100) and (tcp.port == 80)
http and (tcp.port == 443 or tcp.port == 80)
```

### 颜色规则设置
```bash
绿色: HTTP traffic (http)
蓝色: DNS traffic (dns)
黑色: TCP error (tcp.analysis.flags)
红色: TCP problems (tcp.analysis.retransmission)
黄色: Routing (icmp || arp)
```

## 📖 协议分析案例

### Case 1: HTTP请求分析
**场景**：网站访问缓慢问题诊断

```bash
# 抓包命令
tcpdump -i eth0 -w http_debug.pcap host www.example.com

# Wireshark分析步骤
1. 过滤器: http
2. 查看HTTP请求时间线
3. 分析Time列，查找延迟点
```

**关键指标**：
- **DNS解析时间**：查看DNS查询到响应的时间差
- **TCP连接建立时间**：SYN到SYN-ACK的时间差
- **HTTP响应时间**：请求发出到收到响应的时间
- **传输时间**：大文件下载的数据传输效率

**分析示例**：
```
Frame 1: DNS Query for www.example.com
Frame 5: DNS Response (45ms later) ← DNS解析偏慢
Frame 6: TCP SYN to 93.184.216.34:80
Frame 7: TCP SYN-ACK (150ms later) ← 网络延迟高
Frame 8: TCP ACK
Frame 9: HTTP GET request
Frame 15: HTTP 200 OK (2.5s later) ← 服务器处理慢
```

### Case 2: TCP连接问题诊断
**场景**：连接经常断开，需要查找原因

```bash
# 专注TCP分析的抓包
tcpdump -i eth0 'tcp and host 192.168.1.100'
```

**Wireshark分析**：
1. **过滤器**：`tcp.stream eq 0`（分析单个TCP流）
2. **图形分析**：Statistics → Flow Graph
3. **专家系统**：Analyze → Expert Information

**常见TCP问题**：
```bash
# 连接重置
tcp.flags.reset == 1

# 重传问题
tcp.analysis.retransmission

# 窗口满问题
tcp.analysis.window_full

# 乱序包
tcp.analysis.out_of_order
```

### Case 3: DNS问题排查
**场景**：域名解析异常，访问网站失败

```bash
# DNS专用抓包
tcpdump -i eth0 'port 53'
```

**分析要点**：
1. **查询类型**：A记录、AAAA记录、CNAME等
2. **响应码**：NXDOMAIN、SERVFAIL、REFUSED
3. **解析时间**：查询到响应的时间差
4. **DNS服务器**：确认使用的DNS服务器地址

**DNS故障示例**：
```
# 正常DNS查询
Query: www.baidu.com A record
Response: 220.181.38.148 (响应时间: 20ms)

# 异常DNS查询
Query: invalid.domain.com A record
Response: NXDOMAIN (域名不存在)

Query: timeout.example.com A record
No response (DNS服务器无响应)
```

## 🚨 故障排查实战案例

### Case 4: 网络连通性问题
**问题描述**：用户报告无法访问内部服务器192.168.1.200

**排查步骤**：
```bash
# 1. ping测试连通性
ping 192.168.1.200

# 2. 如果ping不通，抓包分析
tcpdump -i eth0 'host 192.168.1.200'

# 3. 发起ping测试同时抓包
ping 192.168.1.200
```

**可能的发现**：
```bash
# 情况1: ARP请求无响应
ARP Request: Who has 192.168.1.200? Tell 192.168.1.100
(无ARP响应) ← 目标主机可能离线或网络不通

# 情况2: ICMP不可达
ICMP Destination Unreachable (Host unreachable) ← 路由问题

# 情况3: 请求超时
ICMP Echo Request发送，但无Echo Reply ← 防火墙或主机防火墙阻止
```

### Case 5: 应用层性能问题
**场景**：Web应用响应缓慢，用户体验差

**抓包分析**：
```bash
# 抓取完整的HTTP会话
tcpdump -i eth0 -w web_performance.pcap 'host webserver.company.com'
```

**Wireshark性能分析**：
1. **Statistics → HTTP → Requests**：查看请求分布
2. **Statistics → TCP Stream Graphs**：分析TCP性能
3. **IO Graphs**：查看流量时间分布

**关键指标分析**：
```bash
# HTTP响应时间分布
统计菜单 → HTTP → Load Distribution
查看平均响应时间、最大响应时间

# TCP重传率
tcp.analysis.retransmission
重传率过高(>3%)表示网络质量问题

# TCP窗口大小
tcp.window_size_value
窗口过小会限制传输速度
```

## 🔒 网络安全相关分析

### Case 6: 异常流量检测
**场景**：发现网络中存在可疑流量

```bash
# 抓取所有流量进行分析
tcpdump -i eth0 -w security_analysis.pcap

# 专注外部通信
tcpdump -i eth0 'not (net 192.168.0.0/16 or net 10.0.0.0/8)'
```

**Wireshark安全分析**：
```bash
# 检测扫描行为
tcp.flags.syn == 1 and tcp.flags.ack == 0
# 查看目标端口分布，大量不同端口可能是端口扫描

# 检测异常DNS查询
dns and dns.qry.name contains "suspicious-domain"

# 检测大量数据传输
frame.len > 1400
# 可能的数据外泄行为
```

## 🛠 专业网络设备协议分析

### SNMP协议分析
```bash
# SNMP抓包
tcpdump -i eth0 'port 161 or port 162'

# Wireshark过滤器
snmp
snmp.version == 2  # SNMP v2c
snmp.error-status != 0  # SNMP错误
```

**SNMP分析要点**：
- **GET请求**：设备状态查询
- **SET请求**：配置修改操作
- **TRAP**：设备主动上报的告警
- **错误码**：noSuchName、badValue等

### NETCONF协议分析
```bash
# NETCONF over SSH (端口830)
tcpdump -i eth0 'port 830'

# Wireshark中需要配置SSH解密才能看到NETCONF内容
```

**关键元素**：
- **RPC调用**：get-config、edit-config
- **数据存储**：running、candidate、startup
- **操作结果**：ok、rpc-error

## 📈 性能调优指导

### TCP性能优化分析
```bash
# TCP窗口缩放
tcp.options.wscale

# TCP时间戳
tcp.options.timestamp

# 选择性确认
tcp.options.sack_perm
```

**优化建议**：
1. **窗口大小**：确保接收窗口不是瓶颈
2. **MSS设置**：避免IP分片
3. **拥塞控制**：观察cwnd变化
4. **延迟优化**：减少往返时间
