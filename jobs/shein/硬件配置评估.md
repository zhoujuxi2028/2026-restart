## ⚡ 性能优化建议

### 1. 内存优化

**关闭不必要的应用**:
```bash
# 检查内存占用大户
top -o MEM | head -n 20

# 建议关闭：
# - 不必要的浏览器标签页
# - 未使用的 IDE
# - 后台应用
```

**预期效果**: 释放 2-4GB 内存

---

### 2. Docker 资源限制

**为容器设置资源限制**:
```yaml
# k8s-manifests/deployment.yaml
resources:
  requests:
    memory: "128Mi"
    cpu: "100m"
  limits:
    memory: "256Mi"
    cpu: "200m"
```

**预期效果**: 防止单个 Pod 占用过多资源

---

### 3. 监控资源使用

**实时监控脚本**:
```bash
# 创建监控脚本
cat > /Users/michaelzhou/Documents/github/2026-restart/jobs/shein/monitor-resources.sh << 'EOF'
#!/bin/bash
echo "=== 系统资源监控 ==="
while true; do
  clear
  echo "时间: $(date '+%H:%M:%S')"
  echo ""
  echo "=== Docker 容器 ==="
  docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"
  echo ""
  echo "=== K8S Pods ==="
  kubectl top pods 2>/dev/null || echo "Metrics Server 未安装"
  echo ""
  echo "=== 系统内存 ==="
  vm_stat | grep -E "Pages free|Pages active" | head -n 2
  sleep 5
done
EOF
chmod +x /Users/michaelzhou/Documents/github/2026-restart/jobs/shein/monitor-resources.sh

# 使用方法
./monitor-resources.sh
```

## ⚠️ 可能遇到的问题

### 问题1：内存不足

**症状**:
```
- Docker Desktop 卡顿
- Pod 启动缓慢
- 系统响应变慢
```

**解决方案**:
```bash
# 1. 减少 Docker 内存分配（4GB → 3GB）
# 2. 减少 Pod 副本数（5 → 3）
# 3. 关闭 Prometheus/Grafana
# 4. 重启 Docker Desktop
```

---

### 问题2：CPU 占用高

**症状**:
```
- 风扇狂转
- 系统发热
- 测试运行缓慢
```

**解决方案**:
```bash
# 1. 限制负载生成器并发数
# 2. 增加测试间隔时间
# 3. 使用串行测试代替并行
```

---

### 问题3：磁盘空间不足

**症状**:
```
- Docker 镜像拉取失败
- 无法创建新容器
```

**解决方案**:
```bash
# 清理 Docker 资源
docker system prune -a --volumes
# 预期释放：5-20GB

# 清理 minikube（如果有）
minikube delete
# 预期释放：5-10GB
```
