# Day 7: Software Testing Methodologies + ISTQB

## Overview
**Focus**: Software Testing Life Cycle (STLC), BDD, OOP in testing, ISTQB fundamentals, Test Strategy & Planning
**Time Allocation**: 3-4 hours
**Difficulty Level**: Intermediate
**Builds On**: All previous days (especially Day 1-2 Cypress, Day 3 API testing)

## Learning Objectives

By the end of Day 7, you should be able to:

1. **Articulate Software Testing Life Cycle (STLC)** in English
   - Six phases from requirement analysis to test closure
   - QA activities in each phase
   - STLC vs SDLC differences
   - Entry/exit criteria for each phase

2. **Explain Behavior-Driven Development (BDD)**
   - Gherkin syntax (Given-When-Then)
   - Cucumber/SpecFlow basics
   - Converting business requirements to executable tests
   - Benefits of BDD for collaboration

3. **Discuss OOP principles in test automation**
   - Page Object Model (POM) pattern
   - Inheritance, encapsulation, polymorphism in tests
   - Design patterns: Factory, Singleton, Strategy
   - Writing maintainable test frameworks

4. **Demonstrate ISTQB knowledge**
   - Testing fundamentals (black-box, white-box, gray-box)
   - Test design techniques (equivalence partitioning, boundary value analysis)
   - Test levels (unit, integration, system, acceptance)
   - Test types (functional, non-functional, regression)

5. **Explain test strategy and planning**
   - Risk-based testing
   - Test pyramid (Unit > Integration > E2E)
   - Test coverage and metrics
   - Defect management lifecycle
