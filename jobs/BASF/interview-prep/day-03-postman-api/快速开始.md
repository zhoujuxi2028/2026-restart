## 🔑 核心学习点 - Authentication Flow

### 认证流程 4 步骤

```
1. Login (POST /login)
   ↓ 获取 token

2. Save Token (pm.collectionVariables.set())
   ↓ 保存到变量

3. Use Token (Authorization: Bearer {{bearerToken}})
   ↓ 在后续请求中使用

4. Test Errors (401 vs 403)
   ↓ 测试错误场景
```

### 重要概念：401 vs 403

| 状态码 | 意思 | 原因 | 解决方案 |
|--------|------|------|----------|
| **401 Unauthorized** | "你是谁？" | 认证失败（没有token或token无效） | 登录获取token |
| **403 Forbidden** | "我知道你是谁，但你不能这么做" | 授权失败（有token但权限不足） | 需要更高权限 |

## 💡 面试回答模板

### Q: "你如何处理 API 认证测试？"

**你的回答：**
> "我实现完整的认证流程测试。首先，发送 POST 请求到 /login 获取 JWT token，使用 pm.collectionVariables.set() 将 token 保存到集合变量中。然后，所有受保护的端点自动从 {{bearerToken}} 变量获取 token，包含在 Authorization header 中。
>
> 我测试正面场景（有效token）和负面场景（无token返回401，有效token但权限不足返回403）。这种方法可扩展 - 我可以运行100个请求但只需要登录一次。"

### Q: "401 和 403 有什么区别？"

**你的回答：**
> "401 Unauthorized 表示'你是谁？'- 认证失败，缺少或无效的 token。403 Forbidden 表示'我知道你是谁，但你不能这么做'- 授权失败，token有效但权限不足。
>
> 例如，普通用户访问管理员端点会得到 403，而不是 401，因为系统知道你的身份，只是你没有足够的权限。"

## 🆘 遇到问题？

### 问题：请求返回 403 Forbidden
**原因：** ReqRes.in API 可能有速率限制或 CORS 限制
**解决：**
- 等待几秒后重试
- 检查网络连接
- API 本身的限制，不影响学习流程

### 问题：找不到 token
**解决：** 先运行 `1️⃣ Login - Valid Credentials` 请求来获取并保存 token

### 问题：测试失败
**检查：**
1. Collection 变量是否已设置（点击集合 → Variables 查看）
2. 网络连接是否正常
3. 请求顺序是否正确（先 Login 再其他）
